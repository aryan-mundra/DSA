Q1 — Longest Even-Length Word in a Sentence

JAVA

Given a sentence, return the longest word whose length is even.
Example: "Be not afraid of greatness, some are born great, some achieve greatness, and some have greatness thrust upon them." → afraid

Intuition

What we need: Find the longest word that has an even number of letters (2, 4, 6, 8, etc.).

The challenge: Words might have punctuation (commas, periods) that we need to remove first, since we only count letters.

How It Works (Step-by-Step)

  1. Split the sentence into words: Break at spaces to get individual words
  2. For each word:
    • Remove all non-letter characters (punctuation, numbers, etc.)
    • Check if the cleaned word has an even length (length % 2 == 0)
    • If it's even AND longer than our current longest, save it
  3. Return the longest even-length word we found

Example walkthrough:

  • "Be" → length 2 (even) → longest = "Be"
  • "not" → length 3 (odd) → skip
  • "afraid" → length 6 (even), longer than "Be" (2) → longest = "afraid"
  • "of" → length 2 (even), not longer than "afraid" (6) → keep "afraid"
  • ... continue checking all words ...
  • Final answer: "afraid" (or "greatness" both length 9... wait, 9 is odd!)

Note: In the example, "afraid" (6 letters) would actually win as the longest even-length word.

public String findWord(String sentence) {
    // split("\\s+") splits the sentence on one or more whitespace characters.
    // e.g. "Be not afraid of greatness," → ["Be", "not", "afraid", "of", "greatness,"]
    String[] words = sentence.split("\\s+");
    String longest = "";

    for (String word : words) {
        // replaceAll("[^a-zA-Z]", "") removes every character that is NOT a letter.
        // e.g. "greatness," → "greatness"  |  "them." → "them"
        String clean = word.replaceAll("[^a-zA-Z]", "");
        if (clean.length() % 2 == 0 && clean.length() > longest.length()) {
            longest = clean;
        }
    }
    return longest;
}
Time: O(n) where n = number of characters  |  Space: O(n)

Q2 — Pair with Maximum Product

JAVA

Given an array with both positive and negative integers, return the pair with the highest product.
Examples: {1,8,3,6,7,0} → {8,7}  |  {-1,-3,-6,3,0,-5} → {-6,-5}

Intuition

The key insight: When multiplying two numbers, the biggest product comes from either:

  • Two large positive numbers (e.g., 8 × 7 = 56)
  • Two very negative numbers (e.g., -6 × -5 = 30, because negative × negative = positive!)

We need to check BOTH possibilities and pick whichever gives the larger product.

Why Sorting Helps

If we sort the array, all the extreme values are at the ends:

  • Leftmost (start): The most negative numbers (smallest values)
  • Rightmost (end): The largest positive numbers

After sorting [-6, -5, -3, -1, 0, 3], we only need to check:

  • First two: -6 × -5 = 30
  • Last two: 3 × 0 = 0
  • Winner: {-6, -5} with product 30

Step-by-Step Algorithm

  1. Sort the array (smallest to largest)
  2. Calculate two candidates:
    • Product of the two largest (rightmost) elements
    • Product of the two smallest (leftmost) elements
  3. Return whichever pair gives the bigger product

Why use long for products? Multiplying two large integers might overflow an int. Using long prevents this.

import java.util.Arrays;

public static int[] findMaxProductPair(int[] arr) {
    if (arr.length < 2) throw new IllegalArgumentException("Need at least 2 elements");

    Arrays.sort(arr);
    int n = arr.length;

    // candidate 1: two largest (rightmost)
    long prodHigh = (long) arr[n - 1] * arr[n - 2];
    // candidate 2: two smallest / most-negative (leftmost)
    long prodLow  = (long) arr[0] * arr[1];

    if (prodHigh >= prodLow) {
        return new int[]{ arr[n - 1], arr[n - 2] };
    } else {
        return new int[]{ arr[0], arr[1] };
    }
}
Time: O(n log n) for sorting  |  Space: O(1) extra

O(n) Approach — Single Pass (No Sorting)

Can we do better than O(n log n)? Yes! We don't actually need to sort the entire array. We only care about 4 numbers:

  • The two largest values (max1, max2)
  • The two smallest values (min1, min2)

We can find these in a single pass through the array!

How It Works

As we scan through each number:

  • If it's bigger than max1, shift max1 → max2, and update max1
  • Else if it's bigger than max2, update max2
  • Same logic for tracking min1 and min2 (the two smallest)

At the end, compare max1 × max2 vs min1 × min2 and return the better pair.

Time: O(n) — much faster!  |  Space: O(1) — only 4 variables

public static int[] findMaxProductPairOptimal(int[] arr) {
    if (arr.length < 2) throw new IllegalArgumentException("Need at least 2 elements");

    int max1 = Integer.MIN_VALUE, max2 = Integer.MIN_VALUE;
    int min1 = Integer.MAX_VALUE, min2 = Integer.MAX_VALUE;

    for (int val : arr) {
        if (val >= max1) {
            max2 = max1;
            max1 = val;
        } else if (val > max2) {
            max2 = val;
        }

        if (val <= min1) {
            min2 = min1;
            min1 = val;
        } else if (val < min2) {
            min2 = val;
        }
    }

    if ((long) max1 * max2 >= (long) min1 * min2) {
        return new int[]{ max1, max2 };
    } else {
        return new int[]{ min1, min2 };
    }
}
Time: O(n) — single pass  |  Space: O(1)

Q3 — Distribute Bananas Equally Among Monkeys

JAVA

Distribute N bananas equally among some monkeys where: each monkey gets > 1 banana, one monkey cannot get all, all bananas must be used, integral distribution only.
Examples: N=2 → false  |  N=15 → true (3 monkeys × 5 bananas)

Intuition

The problem in simple terms: Can we split N bananas equally among some monkeys where:

  • Each monkey gets at least 2 bananas (more than 1)
  • At least 2 monkeys are needed (one monkey can't get all the bananas)
  • The split must be equal (every monkey gets the same amount)

Mathematical Translation

We're looking for two numbers that multiply to give N:

  • Number of monkeys × bananas per monkey = N
  • Both numbers must be ≥ 2

For example: N = 15

  • 3 monkeys × 5 bananas = 15 ✓ (both ≥ 2)
  • 5 monkeys × 3 bananas = 15 ✓ (both ≥ 2)
  • 15 monkeys × 1 banana = 15 ✗ (violates "each gets > 1")
  • 1 monkey × 15 bananas = 15 ✗ (violates "one monkey can't get all")

The Algorithm

Key insight: We're checking if N has any divisor d where both d and N/d are ≥ 2.

  1. Quick rejection: If N < 4, it's impossible (we need at least 2 monkeys × 2 bananas = 4)
  2. Try divisors from 2 up to √N:
    • For each divisor d, check if N/d ≥ 2
    • If yes, we found a valid split!
  3. Why stop at √N? If d divides N, then so does N/d. We don't need to check both; checking up to √N covers all cases.

When It Returns False

  • N < 4: Too few bananas (2 → can't split equally with each getting ≥2; 3 → can't split equally at all)
  • Prime numbers: A prime like 7 only divides by 1 and 7. We'd get 1×7 or 7×1, both violate our rules.
  • Example: N = 7 → divisors are {1, 7} → only split is 1 monkey × 7 bananas (violates rules) → false
static boolean distribute(int N) {
    if (N < 4) return false;

    for (int d = 2; d * d <= N; d++) {
        if (N % d == 0) {
            int bananasPerMonkey = N / d;
            // d starts at 2, so d >= 2 is always true; just check bananasPerMonkey
            if (bananasPerMonkey >= 2) return true;
        }
    }
    return false;
}
Time: O(√N)  |  Space: O(1)
Returns false only for N < 4 and prime numbers (no valid split exists).

Q4 — Employee Hierarchy — Print by Level

JAVA

Given an Employee–Manager table, print all employees grouped by their level in the hierarchy using BFS.
CEO is at level 1, direct reports at level 2, etc.

Intuition

The goal: Print employees organized by their level in the company hierarchy (CEO at level 1, their direct reports at level 2, and so on).

Think of it like a family tree: CEO is the root, their direct reports are their "children," and so on down the tree.

Why Use BFS (Breadth-First Search)?

BFS visits nodes level by level, which is exactly what we need! It processes all employees at level 1, then all at level 2, etc.

Imagine a tree:

        Alice (CEO)
       /   |   \
     Bob  Eve  Frank
    /  \
  Grace Hank

BFS visits: Alice → then Bob, Eve, Frank → then Grace, Hank. This naturally groups by level!

The Algorithm (Two Phases)

Phase 1: Build the Data Structure

  1. Create a "reports" map: For each manager, store a list of their direct reports
    • Example: Alice → [Bob, Eve, Frank]
  2. Track all employees: Keep a set of everyone mentioned
  3. Track who has a manager: Keep a set of people who report to someone
  4. Find the CEO: The CEO is the only person NOT in the "hasManager" set (they don't report to anyone)

Phase 2: BFS Level-by-Level

  1. Start with CEO in the queue
  2. For each level:
    • Remember how many people are in the queue right now (that's the current level's size)
    • Process exactly that many people:
      • Remove them from queue (poll)
      • Add them to the current level's output list
      • Find their direct reports and add them to the queue (they'll be processed in the next level)
    • Sort and print the current level
  3. Repeat until queue is empty (all levels processed)

Why It Works

The key trick is capturing the queue size at the start of each level. This tells us exactly how many people belong to this level. We process only that many, ensuring level boundaries stay correct even as we add next-level employees to the queue.

import java.util.*;

public static void print(Map<String, String> employeeToManager) {
    // Build adjacency list: manager -> list of direct reports
    Map<String, List<String>> reports = new HashMap<>();
    Set<String> allEmployees = new HashSet<>();
    Set<String> hasManager = new HashSet<>();

    for (Map.Entry<String, String> e : employeeToManager.entrySet()) {
        String emp = e.getKey();
        String mgr = e.getValue();
        allEmployees.add(emp);
        allEmployees.add(mgr);
        hasManager.add(emp);
        if (!reports.containsKey(mgr)) {
            reports.put(mgr, new ArrayList<>());
        }
        reports.get(mgr).add(emp);
    }

    // CEO = employee with no manager
    String ceo = null;
    for (String name : allEmployees) {
        if (!hasManager.contains(name)) { ceo = name; break; }
    }

    // BFS level-by-level
    // reports map structure: Map<String, List<String>> reports = new HashMap<>();
    // Example: {"Alice" -> ["Bob", "Charlie", "David"], "Bob" -> ["Eve", "Frank", "Grace"],
    //           "Charlie" -> ["Helen", "Irene", "Jack"], "David" -> ["Karen", "Larry", "Mike"], ...}
    Queue<String> queue = new LinkedList<>();
    queue.add(ceo);
    int level = 1;

    while (!queue.isEmpty()) {
        int size = queue.size();
        List<String> currentLevel = new ArrayList<>();

        for (int i = 0; i < size; i++) {
            String person = queue.poll();
            currentLevel.add(person);
            List<String> directReports = reports.getOrDefault(person, Collections.emptyList());
            queue.addAll(directReports);
        }

        System.out.println("Level " + level + ": " + String.join(" ", currentLevel));
        level++;
    }
}
Time: O(n log n) due to sorting names at each level  |  Space: O(n)

Q5 — Order Search by Time Range

JAVA

Load millions of Order objects (orderId, entryTime). Efficiently find all orders entered between a start and end time.

Intuition

The challenge: We have millions of orders, each with a timestamp. We need to quickly find all orders within a time range (e.g., "Give me all orders from 9:00 to 11:00").

Why not just use an ArrayList? Searching a list linearly would take O(n) time — checking every single order. With millions of orders, this is way too slow!

The Solution: TreeMap (Sorted Map)

What is a TreeMap? It's like a HashMap, but it keeps keys sorted. Think of it as a phone book — names are alphabetically ordered, so you can quickly find a range of names.

Key insight: If we organize orders by time in a TreeMap, we can use binary search to jump directly to a time range instead of scanning everything.

Data Structure Breakdown

  • Key: LocalTime (e.g., 10:15, 9:30)
  • Value: List of all orders placed at that exact time
  • Automatically sorted: TreeMap keeps times in order (7:00 → 8:00 → 9:00 → ...)

Example:

TreeMap {
  7:30  → [Order O314],
  9:15  → [Order O345, Order O999],
  10:15 → [Order O124],
  13:30 → [Order O873]
}

How Searching Works

  1. Call subMap(9:00, 11:00): TreeMap uses binary search to find the boundaries in O(log n) time
  2. Returns a "view": A slice of the map containing only times in [9:00, 11:00]
    • In our example: {9:15 → [...], 10:15 → [...]}
  3. Iterate the view: Collect all orders from those time slots — O(k) where k = number of results

Why This is Fast

Total time: O(log n + k)

  • O(log n) to find range boundaries using binary search in the sorted tree
  • O(k) to collect results (where k = number of matching orders)

Compare to O(n) linear scan — for a million orders, this is 1000× faster!

Why Not HashMap?

Regular HashMap is unordered — you can't ask "give me all keys between 9:00 and 11:00." You'd have to check every single key. TreeMap's sorted property makes range queries possible.

import java.time.LocalTime;
import java.util.*;

class Order {
    String orderId;
    LocalTime entryTime;

    public Order(String orderId, LocalTime entryTime) {
        this.orderId = orderId;
        this.entryTime = entryTime;
    }
}

class OrderStore {
    private TreeMap<LocalTime, List<String>> ordersByTime = new TreeMap<>();

    public void addOrder(Order order) {
        if (!ordersByTime.containsKey(order.entryTime)) {
            ordersByTime.put(order.entryTime, new ArrayList<>());
        }
        ordersByTime.get(order.entryTime).add(order.orderId);
    }

    public List<String> search(LocalTime startTime, LocalTime endTime) {
        List<String> result = new ArrayList<>();

        // subMap returns a view of entries with keys in [startTime, endTime]
        NavigableMap<LocalTime, List<String>> subMap =
            ordersByTime.subMap(startTime, true, endTime, true);

        for (List<String> orders : subMap.values()) {
            result.addAll(orders);
        }
        return result;
    }
}
3. Time Complexity: O(log n + k) where n = total distinct times, k = number of orders in the range.
Finding the range boundaries is O(log n); iterating results is O(k).

Q6 — Database Schema for Online Shopping

SQL

Design tables for: customers, orders, order items, shipping addresses. Specify PKs, FKs, indexes, and write two queries.

Schema Design

  • customers — one row per customer
  • shipping_addresses — multiple addresses per customer
  • orders — each order belongs to one customer, shipped to one address
  • order_items — multiple items per order
-- Create the customers table to store basic customer information
CREATE TABLE customers (
    customer_id   INT          PRIMARY KEY AUTO_INCREMENT,  -- Unique ID for each customer, auto-generates next number
    name          VARCHAR(100) NOT NULL,                -- Customer's full name, cannot be empty
    email         VARCHAR(150) NOT NULL UNIQUE,          -- Email address, must be unique (no duplicates), cannot be empty
    phone         VARCHAR(20)                               -- Phone number, optional (can be NULL)
);

-- Create the shipping_addresses table to store multiple delivery addresses for each customer
CREATE TABLE shipping_addresses (
    address_id    INT          PRIMARY KEY AUTO_INCREMENT,  -- Unique ID for each address
    customer_id   INT          NOT NULL,                -- Links to customers table (which customer owns this address)
    street        VARCHAR(255),                             -- Street address
    city          VARCHAR(100),                             -- City name
    state         VARCHAR(100),                             -- State/province
    zip_code      VARCHAR(20),                              -- Postal/ZIP code
    country       VARCHAR(100),                             -- Country name
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)  -- Ensures customer_id exists in customers table
);

-- Create the orders table to store each customer's orders
CREATE TABLE orders (
    order_id      INT          PRIMARY KEY AUTO_INCREMENT,  -- Unique ID for each order
    customer_id   INT          NOT NULL,                -- Links to customers table (who placed this order)
    address_id    INT          NOT NULL,                -- Links to shipping_addresses table (where to ship this order)
    order_date    DATETIME     DEFAULT CURRENT_TIMESTAMP,  -- When the order was placed; automatically set to current date/time
    total_amount  DECIMAL(10,2),                             -- Total price (10 digits, 2 after decimal: e.g., 12345678.99)
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id),    -- Ensures customer_id exists
    FOREIGN KEY (address_id)  REFERENCES shipping_addresses(address_id)  -- Ensures address_id exists
);

-- Create the order_items table to store individual products within each order
CREATE TABLE order_items (
    item_id       INT          PRIMARY KEY AUTO_INCREMENT,  -- Unique ID for each line item
    order_id      INT          NOT NULL,                -- Links to orders table (which order this item belongs to)
    product_name  VARCHAR(200),                             -- Name of the product
    quantity      INT          NOT NULL DEFAULT 1,        -- How many of this item (defaults to 1 if not specified)
    price         DECIMAL(10,2),                             -- Price per unit
    FOREIGN KEY (order_id) REFERENCES orders(order_id)           -- Ensures order_id exists in orders table
);

-- ═══ Indexes for faster lookups ═══
-- Index on orders.customer_id: speeds up queries like "find all orders for customer X"
CREATE INDEX idx_orders_customer    ON orders(customer_id);
-- Index on shipping_addresses.customer_id: speeds up "find all addresses for customer X"
CREATE INDEX idx_addresses_customer ON shipping_addresses(customer_id);
-- Index on order_items.order_id: speeds up "find all items in order X"
CREATE INDEX idx_items_order        ON order_items(order_id);

Query 1 — Customers who have NOT placed any order

-- Find customers who have never placed an order
SELECT c.name, c.phone              -- Get customer name and phone
FROM   customers c                  -- Start with all customers (alias 'c')
LEFT JOIN orders o ON c.customer_id = o.customer_id  -- Try to match each customer to their orders
                                      -- LEFT JOIN keeps ALL customers, even if no match in orders
WHERE  o.order_id IS NULL;           -- Filter: keep only rows where order_id is NULL (no order found)
                                      -- If order_id is NULL, it means this customer has 0 orders

Query 2 — Customers with a single order containing 5+ items

-- Find customers who have at least one order with 5 or more items in it
SELECT DISTINCT c.name, c.email     -- Get unique customer names and emails (DISTINCT removes duplicates)
FROM   customers c                  -- Start with customers table
JOIN   orders o      ON c.customer_id = o.customer_id  -- Match customers to their orders
JOIN   order_items oi ON o.order_id = oi.order_id      -- Match orders to their items
GROUP BY c.customer_id, c.name, c.email, o.order_id  -- Group rows by customer AND order
                                      -- This creates one group per order, so we can count items per order
HAVING   COUNT(oi.item_id) >= 5;   -- Filter groups: keep only orders with 5+ items
                                      -- HAVING is like WHERE, but works on grouped data

Q7 — LRU Cache Implementation

JAVA

Implement an LRU Cache with O(1) get and put using a HashMap + Doubly Linked List.

Intuition

What is LRU? "Least Recently Used" — a caching strategy where we evict the item that hasn't been accessed for the longest time when the cache is full.

Real-world example: Your browser cache. If your cache can hold 100 pages and you visit a 101st page, the browser deletes the page you visited longest ago.

The Challenge

We need both operations to be O(1) (instant):

  • get(key): Find and return the value
  • put(key, value): Insert or update, and if full, remove the least recently used item

One data structure alone can't do this:

  • HashMap: O(1) lookup, but doesn't track access order
  • Linked List: Can track order, but O(n) to find a specific item

Solution: Use BOTH together!

The Hybrid Data Structure

1. HashMap<key, Node>: Gives us O(1) lookup to find any node instantly

2. Doubly Linked List: Maintains access order

  • Head (most recent): Items just accessed
  • Tail (least recent): Items accessed longest ago
head ←→ [recently used] ←→ [used earlier] ←→ [oldest] ←→ tail
         (keep these)                        (evict this if full)

How Operations Work

get(key):

  1. Look up key in HashMap → get the node instantly (O(1))
  2. If not found, return -1
  3. If found: move the node to the front of the list (it's now most recently used)
    • Remove it from current position
    • Insert right after head
  4. Return the value

put(key, value):

  1. If key exists:
    • Update its value
    • Move it to the front (most recently used)
  2. If key is new:
    • If cache is full, remove the node at the tail (least recently used)
    • Create new node and insert at the front
    • Add to HashMap

Why Doubly Linked?

We need prev and next pointers to remove a node in O(1) without scanning. To remove a node, we update:

  • node.prev.next = node.next
  • node.next.prev = node.prev

Single-linked list would require O(n) scan to find the previous node.

Sentinel Nodes (head/tail)

We use dummy head and tail nodes to avoid null-pointer checks. They never store real data; they just mark boundaries.

import java.util.HashMap;
import java.util.Map;

class LRUCache {

    private class Node {
        int key, value;
        Node prev, next;
        Node(int k, int v) { key = k; value = v; }
    }

    private final int capacity;
    private final Map<Integer, Node> map;
    private final Node head, tail; // sentinel nodes

    public LRUCache(int capacity) {
        this.capacity = capacity;
        map = new HashMap<>();
        head = new Node(0, 0);
        tail = new Node(0, 0);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        remove(node);
        insertAfterHead(node);
        return node.value;
    }

    public void put(int key, int value) {
        if (map.containsKey(key)) {
            Node node = map.get(key);
            node.value = value;
            remove(node);
            insertAfterHead(node);
        } else {
            if (map.size() == capacity) {
                Node lru = tail.prev;
                remove(lru);
                map.remove(lru.key);
            }
            Node newNode = new Node(key, value);
            map.put(key, newNode);
            insertAfterHead(newNode);
        }
    }

    private void remove(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void insertAfterHead(Node node) {
        node.next = head.next;
        node.prev = head;
        head.next.prev = node;
        head.next = node;
    }
}
Time: O(1) for both get and put  |  Space: O(capacity)
