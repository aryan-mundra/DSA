01—Sliding Window

A moving window traverses an array or string while reusing previous computations, turning expensive brute-force scans into efficient linear-time algorithms.

When to use

  • Longest / shortest substring or subarray
  • Maximum sum / average over a fixed window
  • Minimum window containing a pattern
  • "At most K" distinct / "exactly K" counting

Interview Signal

You find yourself recomputing overlapping ranges, or the problem says contiguous, longest, shortest, or at most K. (If sums involve negative numbers, switch to Prefix Sum + HashMap — shrinking can increase a sum.)

Templates

Rule #1 — Fixed Window (Size K)

Problem. Find the maximum sum of any contiguous subarray of size k.

Input: nums = [2,1,5,1,3,2], k = 3 Output: 9 (subarray [5,1,3])
int left = 0;
int sum = 0;
int maxSum = 0;
for (int right = 0; right < n; right++) {
    sum += arr[right];                 // expand
    if ((right - left + 1) > k) {      // too big
        sum -= arr[left];
        left++;
    }
    if (right - left + 1 == k) {   // exact size
        maxSum = Math.max(maxSum, sum);
    }
}
return maxSum;

Time O(N)  ·  Space O(1)

Rule #2 — Longest Subarray (update AFTER shrink)

Problem. Longest substring with at most k distinct characters.

Input: s = "eceba", k = 2 Output: 3 ("ece")
int left = 0;
int best = 0;
for (int right = 0; right < n; right++) {
    add(arr[right]);                   // expand
    while (!isValid()) {               // shrink until valid
        remove(arr[left]);
        left++;
    }
    best = Math.max(best, right - left + 1);   // update AFTER shrink
}
return best;

Time O(N)  ·  Space O(K)

Rule #3 — Shortest Subarray (update DURING shrink)

Problem. Smallest length of a contiguous subarray with sum >= target (all positive).

Input: target = 7, nums = [2,3,1,2,4,3] Output: 2 (subarray [4,3])
int left = 0;
int best = Integer.MAX_VALUE;
for (int right = 0; right < n; right++) {
    add(arr[right]);
    while (isValid()) {                            // still valid
        best = Math.min(best, right - left + 1);   // update DURING shrink
        remove(arr[left]);
        left++;
    }
}
return best == Integer.MAX_VALUE ? 0 : best;

Time O(N)  ·  Space O(1)

Rule #4 — "Exactly K" = atMost(K) − atMost(K−1)

Problem. Count the subarrays that contain exactly k distinct integers.

Input: nums = [1,2,1,2,3], k = 2 Output: 7
int exactlyK(int[] arr, int k) {
    return atMostK(arr, k) - atMostK(arr, k - 1);
}
int atMostK(int[] arr, int k) {
    Map<Integer,Integer> freq = new HashMap<>();
    int left = 0;
    int count = 0;
    for (int right = 0; right < arr.length; right++) {
        freq.merge(arr[right], 1, Integer::sum);
        while (freq.size() > k) {
            if (freq.merge(arr[left], -1, Integer::sum) == 0) {
                freq.remove(arr[left]);
            }
            left++;
        }
        count += right - left + 1;     // subarrays ending at right
    }
    return count;
}

Time O(N)  ·  Space O(K)

Why not just right − left + 1? That counts every sub-window inside a valid window — true for "at most K" (shrinking never adds distinctness) but false for "exactly K", where trailing duplicates create a variable number of invalid sub-windows. No fixed adjustment works, so we subtract two "at most" counts instead.

Rule #5 — Minimum Window Substring (have / need)

Problem. Smallest substring of s that contains every character of t.

Input: s = "ADOBECODEBANC", t = "ABC" Output: "BANC"
Map<Character,Integer> need = new HashMap<>();
Map<Character,Integer> have = new HashMap<>();
for (char c : pattern.toCharArray()) {
    need.merge(c, 1, Integer::sum);
}
int required = need.size();
int formed = 0;
int left = 0;
int minLen = Integer.MAX_VALUE;
int minStart = 0;
for (int right = 0; right < s.length(); right++) {
    char c = s.charAt(right);
    have.merge(c, 1, Integer::sum);
    if (need.containsKey(c) && have.get(c).equals(need.get(c))) {
        formed++;
    }
    while (formed == required) {
        if (right - left + 1 < minLen) {
            minLen = right - left + 1;
            minStart = left;
        }
        char lc = s.charAt(left);
        have.merge(lc, -1, Integer::sum);
        if (need.containsKey(lc) && have.get(lc) < need.get(lc)) {
            formed--;
        }
        left++;
    }
}
return minLen == Integer.MAX_VALUE ? "" : s.substring(minStart, minStart + minLen);

Time O(N)  ·  Space O(K)

Rule #6 — Count All Valid Subarrays

Problem. Count contiguous subarrays whose product is strictly less than k.

Input: nums = [10,5,2,6], k = 100 Output: 8
int left = 0;
long count = 0;
for (int right = 0; right < n; right++) {
    add(arr[right]);
    while (!isValid()) {
        remove(arr[left]);
        left++;
    }
    count += right - left + 1;     // all subarrays ending at right
}
return count;

Time O(N)  ·  Space O(1)

02—Prefix Sum

Precompute cumulative totals once so that any range sum is answered in constant time — the backbone of subarray-sum and range-query problems.

When to use

  • Many range-sum queries on a static array
  • Count subarrays with sum = K (works with negatives)
  • Sum divisible by K (remainder buckets)
  • 2D rectangle sums and range updates

Interview Signal

Repeated range-sum queries, or counting subarrays by sum — especially with negative numbers, where a sliding window cannot work.

Key Idea. Range sum [L,R] = prefix[R] − prefix[L−1]. To count subarrays with sum K, store seen prefix sums in a map and look for prefix − K.
Templates

Rule #1 — Basic Prefix Sum (range queries)

Problem. Answer many range-sum queries sumRange(L, R) on a fixed array.

Input: nums = [-2,0,3,-5,2,-1], sumRange(0, 2) Output: 1 (-2 + 0 + 3)
int[] prefix = new int[n];
prefix[0] = nums[0];
for (int i = 1; i < n; i++) {
    prefix[i] = prefix[i - 1] + nums[i];
}
// sum of [L,R] = prefix[R] - (L > 0 ? prefix[L - 1] : 0)

Build O(N)  ·  Query O(1)

Rule #2 — Prefix Sum + HashMap (count sum = K)

Problem. Count the subarrays whose sum equals k (values may be negative).

Input: nums = [1,1,1], k = 2 Output: 2
Map<Integer,Integer> map = new HashMap<>();
map.put(0, 1);
int prefix = 0;
int count = 0;
for (int x : nums) {
    prefix += x;
    count += map.getOrDefault(prefix - k, 0);
    map.merge(prefix, 1, Integer::sum);
}
return count;

Time O(N)  ·  Space O(N)

Rule #3 — Mod Prefix (sum divisible by K)

Problem. Count the subarrays whose sum is divisible by k.

Input: nums = [4,5,0,-2,-3,1], k = 5 Output: 7
Map<Integer,Integer> map = new HashMap<>();
map.put(0, 1);
int prefix = 0;
int count = 0;
for (int x : nums) {
    prefix += x;
    int rem = ((prefix % k) + k) % k;     // normalise negative remainders
    count += map.getOrDefault(rem, 0);
    map.merge(rem, 1, Integer::sum);
}
return count;

Time O(N)  ·  Space O(K)

Rule #4 — 2D Prefix Sum

Problem. Answer sum queries over any sub-rectangle of a matrix.

Input: matrix = [[3,0,1],[2,1,0],[1,2,3]], region (0,0)-(1,1) Output: 6
// pre[i][j] = sum of rectangle (0,0)..(i-1,j-1)
pre[i][j] = mat[i-1][j-1] + pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1];
// sum of (r1,c1)..(r2,c2):
int s = pre[r2+1][c2+1] - pre[r1][c2+1] - pre[r2+1][c1] + pre[r1][c1];

Build O(M·N)  ·  Query O(1)

Rule #5 — Difference Array (range updates)

Problem. Apply many range-add updates, then return the final array.

Input: n = 5, updates = [[1,3,2],[2,4,3]] Output: [0,2,5,5,3]
int[] diff = new int[n + 1];
for (int[] u : updates) {            // u = [start, end, val]
    diff[u[0]] += u[2];
    diff[u[1] + 1] -= u[2];
}
int[] res = new int[n];
res[0] = diff[0];
for (int i = 1; i < n; i++) {
    res[i] = res[i - 1] + diff[i];
}

Updates O(1) each  ·  Build O(N)

Rule #6 — Prefix × Suffix Product

Problem. Return output[i] = product of all elements except nums[i] (no division).

Input: nums = [1,2,3,4] Output: [24,12,8,6]
int[] res = new int[n];
res[0] = 1;
for (int i = 1; i < n; i++) {
    res[i] = res[i - 1] * nums[i - 1];   // prefix
}
int right = 1;
for (int i = n - 1; i >= 0; i--) {
    res[i] *= right;
    right *= nums[i];
}   // suffix
return res;

Time O(N)  ·  Space O(1) extra

03—Binary Search

Repeatedly halve a sorted or monotonic search space, homing in on the answer in O(log N) comparisons instead of scanning everything.

When to use

  • Rotated arrays and peak finding

The data is sorted or monotonic, or the phrase minimize the maximum / maximize the minimum appears — a hint to binary search the answer, not an array.

Templates

Rule #1 — Classic Binary Search

Problem. Return the index of target in a sorted array, or -1.

Input: nums = [-1,0,3,5,9,12], target = 9 Output: 4
int left = 0;
int right = arr.length - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] < target) {
        left = mid + 1;
    } else {
        right = mid - 1;
    }
}
return -1;

Rule #2 — First / Last Occurrence

Problem. Find the first and last index of target in a sorted array with duplicates.

Input: nums = [5,7,7,8,8,10], target = 8 Output: [3,4]
// First: when found, record and keep going LEFT
if (arr[mid] == target) {
    result = mid;
    right = mid - 1;
}
// Last:  when found, record and keep going RIGHT
if (arr[mid] == target) {
    result = mid;
    left = mid + 1;
}

Rule #3 — Rotated Sorted Array

Problem. Search for target in a rotated sorted array.

Input: nums = [4,5,6,7,0,1,2], target = 0 Output: 4
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) {
        return mid;
    }
    if (nums[left] <= nums[mid]) {                  // left half sorted
        if (nums[left] <= target && target < nums[mid]) {
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    } else {                                        // right half sorted
        if (nums[mid] < target && target <= nums[right]) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
}

Rule #4 — Find Peak Element

Problem. Return the index of any peak element (greater than its neighbours).

Input: nums = [1,2,3,1] Output: 2
while (left < right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] < nums[mid + 1]) {
        left = mid + 1;   // peak on the right
    } else {
        right = mid;
    }
}
return left;

Rule #5 — Binary Search on the Answer

Problem. Minimum eating speed so Koko finishes all banana piles within h hours.

Input: piles = [3,6,7,11], h = 8 Output: 4
// MINIMIZE (e.g. minimize max, minimize speed)
int left = lo;
int right = hi;
while (left < right) {
    int mid = left + (right - left) / 2;
    if (isFeasible(mid)) {
        right = mid;        // feasible -> try smaller
    } else {
        left = mid + 1;
    }
}
return left;

// MAXIMIZE (e.g. maximize min, maximize capacity)
int left = lo;
int right = hi;
while (left < right) {
    int mid = left + (right - left + 1) / 2;  // ceil mid (round UP)
    if (isFeasible(mid)) {
        left = mid;          // feasible -> try larger
    } else {
        right = mid - 1;
    }
}
return left;
// Why ceil mid here? This branch does left = mid (not mid + 1). With floor mid,
// when right = left + 1 you'd get mid = left, set left = mid = left, and loop
// forever. Rounding UP guarantees mid moves, avoiding the infinite loop.

Time O(N · log range)  ·  Space O(1)

Rule #6 — Search a 2D Matrix

Problem. Search a target in a row- and column-sorted matrix.

Input: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 3 Output: true
// fully sorted: index as row = mid / cols, col = mid % cols
// staircase search (rows & cols sorted):
int row = 0;
int col = n - 1;
while (row < m && col >= 0) {
    if (matrix[row][col] == target) {
        return true;
    } else if (matrix[row][col] < target) {
        row++;
    } else {
        col--;
    }
}

while (left <= right)Find an exact element

Common mistakes

✗ Overflow — always mid = left + (right − left) / 2.

✗ Failing to spot the monotonic property for "search on answer".

✗ Mixing up minimize (right = mid) and maximize (left = mid with ceil mid).

04—Monotonic Stack

A stack kept in increasing or decreasing order, where each element is pushed and popped at most once — finding the next greater or smaller element in a single linear pass.

When to use

  • Next / previous greater or smaller element
  • Largest rectangle in a histogram
  • Sliding-window maximum (monotonic deque)
  • Building the optimal number (remove K digits)

Interview Signal

"For each element, find the next bigger/smaller one", or anything about spans, histograms, or temperatures.

Templates

Rule #1 — Next Greater Element (decreasing stack)

Problem. For each day, how many days until a warmer temperature (0 if none).

Input: temps = [73,74,75,71,69,72,76,73] Output: [1,1,4,2,1,1,0,0]
int[] result = new int[n];
Arrays.fill(result, -1);
Stack<Integer> stack = new Stack<>();        // stores indices
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && arr[stack.peek()] < arr[i]) {
        int idx = stack.pop();
        result[idx] = i - idx;    // Daily Temperatures: days until warmer
        // result[idx] = arr[i];  // Next Greater Element: the greater VALUE
    }
    stack.push(i);
}

Rule #2 — Largest Rectangle in Histogram

Problem. Largest rectangle area in a histogram of bar heights.

Input: heights = [2,1,5,6,2,3] Output: 10
Stack<Integer> stack = new Stack<>();
int maxArea = 0;
for (int i = 0; i <= n; i++) {
    int cur = (i == n) ? 0 : heights[i];
    while (!stack.isEmpty() && heights[stack.peek()] > cur) {
        int h = heights[stack.pop()];
        int w = stack.isEmpty() ? i : i - stack.peek() - 1;
        maxArea = Math.max(maxArea, h * w);
    }
    stack.push(i);
}

Rule #3 — Sliding Window Maximum (monotonic deque)

Problem. Maximum of every contiguous window of size k.

Input: nums = [1,3,-1,-3,5,3,6,7], k = 3 Output: [3,3,5,5,6,7]
Deque<Integer> dq = new ArrayDeque<>();
int[] result = new int[n - k + 1];
for (int i = 0; i < n; i++) {
    if (!dq.isEmpty() && dq.peekFirst() < i - k + 1) {
        dq.pollFirst();   // drop out-of-window
    }
    while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) {
        dq.pollLast(); // keep decreasing
    }
    dq.offerLast(i);
    if (i >= k - 1) {
        result[i - k + 1] = nums[dq.peekFirst()];
    }
}
return result;

Time O(N)  ·  Space O(K)

Rule #2 — Slow / Fast (Floyd)

Problem. Detect whether a linked list has a cycle.

Input: head = [3,2,0,-4], tail connects to index 1 Output: true
ListNode slow = head;
ListNode fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow == fast) {
        return true;        // cycle found
    }
}
// middle: when fast hits the end, slow is the midpoint

06—Greedy

Make the locally optimal choice at each step and never look back — valid when a greedy choice provably leads to the global optimum.

When to use

  • Interval scheduling — max non-overlapping, merge
  • Jump game · gas station · stock buy/sell
  • Partition labels · task scheduler
  • Fractional knapsack (items can be split)

Interview Signal

"Maximum / minimum / optimal" with a natural ordering, and once you commit to a choice you never need to revisit it.

Classic Problem

Non-Overlapping Intervals (Activity Selection)

Sort by end time, then greedily keep each interval that starts after the last kept end
💡
Key Idea. Sort by the right metric (often end time), then commit to every compatible choice. Finishing early leaves the most room for what follows.
Templates

Rule #1 — Non-Overlapping Intervals (sort by END)

Problem. Minimum number of intervals to remove so the rest don't overlap.

Input: intervals = [[1,2],[2,3],[3,4],[1,3]] Output: 1
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);   // by END
int count = 0;
int lastEnd = Integer.MIN_VALUE;
for (int[] it : intervals) {
    if (it[0] >= lastEnd) {
        count++;
        lastEnd = it[1];
    }
}

Rule #2 — Merge Intervals (sort by START)

Problem. Merge all overlapping intervals.

Input: intervals = [[1,3],[2,6],[8,10],[15,18]] Output: [[1,6],[8,10],[15,18]]
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);   // by START
List<int[]> merged = new ArrayList<>();
for (int[] it : intervals) {
    if (merged.isEmpty() || it[0] > merged.get(merged.size() - 1)[1]) {
        merged.add(it);
    } else {
        merged.get(merged.size() - 1)[1] =
            Math.max(merged.get(merged.size() - 1)[1], it[1]);
    }
}

Rule #3 — Minimum Meeting Rooms

Problem. Minimum number of meeting rooms required.

Arrays.sort(ends);

Rule #4 — Jump Game (track farthest reach)

Rule #9 — Fractional Knapsack

Problem. Maximum value within a weight capacity; items may be split.

    Double.compare((double) b[1] / b[0], (double) a[1] / a[0]));   // ratio desc
double value = 0;

    if (it[0] <= rem) {
        value += it[1];
        rem -= it[0];

        value += it[1] * ((double) rem / it[0]);

07—Recursion & Backtracking

Solve a problem by reducing it to smaller copies of itself; backtracking adds an "undo" step so every valid configuration can be explored.

When to use

  • Generate all subsets / combinations / permutations
  • Constraint puzzles — N-Queens, word search
  • Divide and conquer — merge sort

Interview Signal

"Generate all", "find every combination", or a search through choices with dead-ends to recover from.

Classic Problem

Subsets

Subsets of [1, 2, 3] — at each element, branch exclude (left) or include (right)
💡
Key Idea. Choose → Explore → Unchoose. Define the base case first, make a move, recurse, then undo the move and try the next option.
Templates

Rule #1 — Subsets (include or exclude)

Problem. Generate all subsets of a set of distinct integers.

Input: nums = [1,2,3] Output: [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]
void subsets(int[] nums, int i, List<Integer> cur, List<List<Integer>> res) {
    if (i == nums.length) {
        res.add(new ArrayList<>(cur));
        return;
    }
    subsets(nums, i + 1, cur, res);            // exclude
    cur.add(nums[i]);
    subsets(nums, i + 1, cur, res);            // include
    cur.remove(cur.size() - 1);                // backtrack
}

Time O(2ⁿ)  ·  Space O(N)

Rule #2 — Permutations (visited array)

Problem. Generate all permutations of distinct integers.

Input: nums = [1,2,3] Output: 6 perms: [1,2,3],[1,3,2], ... ,[3,2,1]
void permute(int[] nums, List<Integer> cur, boolean[] used, List<List<Integer>> res) {
    if (cur.size() == nums.length) {
        res.add(new ArrayList<>(cur));
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) {
            continue;
        }
        used[i] = true;
        cur.add(nums[i]);
        permute(nums, cur, used, res);
        cur.remove(cur.size() - 1);
        used[i] = false;
    }
}

Time O(N! · N)  ·  Space O(N)

Rule #3 — Combinations (choose K from N)

Problem. All ways to choose k numbers from 1..n.

Input: n = 4, k = 2 Output: [[1,2],[1,3],[1,4],[2,3],[2,4],[3,4]]
void combine(int n, int k, int start, List<Integer> cur, List<List<Integer>> res) {
    if (cur.size() == k) {
        res.add(new ArrayList<>(cur));
        return;
    }
    for (int i = start; i <= n; i++) {
        cur.add(i);
        combine(n, k, i + 1, cur, res);   // i + 1: forward only
        cur.remove(cur.size() - 1);
    }
}

Rule #5 — Divide and Conquer (split → solve → merge)

Problem. Sort an array using merge sort.

Input: arr = [5,2,4,1,3] Output: [1,2,3,4,5]
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) {
        return;
    }
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

08—Top-K / Heap

A heap of size K keeps just the most important K elements live, giving O(N log K) selection without ever sorting the whole dataset.

When to use

  • Kth largest / smallest element
  • K most frequent elements
  • K closest points
  • Merge K sorted lists · streaming data

Interview Signal

You need a handful of top (or bottom) results, not a complete ordering — sorting everything would be wasteful.

Classic Problem

Top K Frequent Elements

Min-heap of size K = 3, keyed by frequency — root is the least frequent of the keepers
💡
Key Idea. For K largest use a min-heap of size K (root = smallest of the keepers, so it's first to evict). For K smallest use a max-heap.
Templates

Rule #1 — K Largest Elements (min-heap of size K)

Problem. Return node values level by level.

    if (p == null && q == null) {
        return true;
    }
    if (p == null || q == null) {
        return false;

        && isSameTree(p.left, q.left)
        && isSameTree(p.right, q.right);

Rule #5 — Path Sum / Max Path Sum

Problem. Is there a root-to-leaf path summing to target?

Input: root = [5,4,8,11,null,13,4,7,2], target = 22 Output: true
boolean hasPathSum(TreeNode root, int target) {
    if (root == null) {
        return false;
    }
    if (root.left == null && root.right == null) {
        return root.val == target;
    }
    return hasPathSum(root.left, target - root.val)
        || hasPathSum(root.right, target - root.val);
}
// max path sum: gain = node.val + max(leftGain, rightGain);

Rule #6 — Lowest Common Ancestor

Problem. Lowest common ancestor of two nodes in a binary tree.

Input: root = [3,5,1,6,2,0,8], p = 5, q = 1 Output: 3
TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) {
        return root;
    }
    TreeNode left = lca(root.left, p, q);
    TreeNode right = lca(root.right, p, q);
    if (left != null && right != null) {
        return root;   // split point
    }
    return left != null ? left : right;
}
// BST: both < root go left, both > root go right, else root is LCA

Rule #7 — BST Operations

Problem. Is the tree a valid binary search tree?

Input: root = [2,1,3] Output: true

    if (node == null) {
        return true;
    }
    if (node.val <= min || node.val >= max) {
        return false;
    }
    return valid(node.left, min, node.val) && valid(node.right, node.val, max);
}
