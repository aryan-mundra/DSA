01—Sliding Window

A moving window traverses an array or string while reusing previous computations, turning expensive brute-force scans into efficient linear-time algorithms.

When to use

  • Longest / shortest substring or subarray
  • Maximum sum / average over a fixed window
  • Minimum window containing a pattern
  • "At most K" distinct / "exactly K" counting

Interview Signal

You find yourself recomputing overlapping ranges, or the problem says contiguous, longest, shortest, or at most K. (If sums involve negative numbers, switch to Prefix Sum + HashMap — shrinking can increase a sum.)

Templates

Rule #1 — Fixed Window (Size K)

Problem. Find the maximum sum of any contiguous subarray of size k.

Input: nums = [2,1,5,1,3,2], k = 3 Output: 9 (subarray [5,1,3])
int left = 0;
int sum = 0;
int maxSum = 0;
for (int right = 0; right < n; right++) {
    sum += arr[right];                 // expand
    if ((right - left + 1) > k) {      // too big
        sum -= arr[left];
        left++;
    }
    if (right - left + 1 == k) {   // exact size
        maxSum = Math.max(maxSum, sum);
    }
}
return maxSum;

Time O(N)  ·  Space O(1)

Rule #2 — Longest Subarray (update AFTER shrink)

Problem. Longest substring with at most k distinct characters.

Input: s = "eceba", k = 2 Output: 3 ("ece")
int left = 0;
int best = 0;
for (int right = 0; right < n; right++) {
    add(arr[right]);                   // expand
    while (!isValid()) {               // shrink until valid
        remove(arr[left]);
        left++;
    }
    best = Math.max(best, right - left + 1);   // update AFTER shrink
}
return best;

Time O(N)  ·  Space O(K)

Rule #3 — Shortest Subarray (update DURING shrink)

Problem. Smallest length of a contiguous subarray with sum >= target (all positive).

Input: target = 7, nums = [2,3,1,2,4,3] Output: 2 (subarray [4,3])
int left = 0;
int best = Integer.MAX_VALUE;
for (int right = 0; right < n; right++) {
    add(arr[right]);
    while (isValid()) {                            // still valid
        best = Math.min(best, right - left + 1);   // update DURING shrink
        remove(arr[left]);
        left++;
    }
}
return best == Integer.MAX_VALUE ? 0 : best;

Time O(N)  ·  Space O(1)

Rule #4 — "Exactly K" = atMost(K) − atMost(K−1)

Problem. Count the subarrays that contain exactly k distinct integers.

Input: nums = [1,2,1,2,3], k = 2 Output: 7
int exactlyK(int[] arr, int k) {
    return atMostK(arr, k) - atMostK(arr, k - 1);
}
int atMostK(int[] arr, int k) {
    Map<Integer,Integer> freq = new HashMap<>();
    int left = 0;
    int count = 0;
    for (int right = 0; right < arr.length; right++) {
        freq.put(arr[right], freq.getOrDefault(arr[right], 0) + 1);
        while (freq.size() > k) {
            int newCount = freq.get(arr[left]) - 1;
            if (newCount == 0) {
                freq.remove(arr[left]);
            } else {
                freq.put(arr[left], newCount);
            }
            left++;
        }
        count += right - left + 1;     // subarrays ending at right
    }
    return count;
}

Time O(N)  ·  Space O(K)

Why not just right − left + 1? That counts every sub-window inside a valid window — true for "at most K" (shrinking never adds distinctness) but false for "exactly K", where trailing duplicates create a variable number of invalid sub-windows. No fixed adjustment works, so we subtract two "at most" counts instead.

Rule #5 — Minimum Window Substring (have / need)

Problem. Smallest substring of s that contains every character of t.

Input: s = "ADOBECODEBANC", t = "ABC" Output: "BANC"
Map<Character,Integer> need = new HashMap<>();
Map<Character,Integer> have = new HashMap<>();
for (char c : pattern.toCharArray()) {
    need.put(c, need.getOrDefault(c, 0) + 1);
}
int required = need.size();
int formed = 0;
int left = 0;
int minLen = Integer.MAX_VALUE;
int minStart = 0;
for (int right = 0; right < s.length(); right++) {
    char c = s.charAt(right);
    have.put(c, have.getOrDefault(c, 0) + 1);
    if (need.containsKey(c) && have.get(c).equals(need.get(c))) {
        formed++;
    }
    while (formed == required) {
        if (right - left + 1 < minLen) {
            minLen = right - left + 1;
            minStart = left;
        }
        char lc = s.charAt(left);
        have.put(lc, have.getOrDefault(lc, 0) - 1);
        if (need.containsKey(lc) && have.get(lc) < need.get(lc)) {
            formed--;
        }
        left++;
    }
}
return minLen == Integer.MAX_VALUE ? "" : s.substring(minStart, minStart + minLen);

Time O(N)  ·  Space O(K)

Rule #6 — Count All Valid Subarrays

Problem. Count contiguous subarrays whose product is strictly less than k.

Input: nums = [10,5,2,6], k = 100 Output: 8
int left = 0;
long count = 0;
for (int right = 0; right < n; right++) {
    add(arr[right]);
    while (!isValid()) {
        remove(arr[left]);
        left++;
    }
    count += right - left + 1;     // all subarrays ending at right
}
return count;

Time O(N)  ·  Space O(1)

02—Prefix Sum

Precompute cumulative totals once so that any range sum is answered in constant time — the backbone of subarray-sum and range-query problems.

When to use

  • Many range-sum queries on a static array
  • Count subarrays with sum = K (works with negatives)
  • Sum divisible by K (remainder buckets)
  • 2D rectangle sums and range updates

Interview Signal

Repeated range-sum queries, or counting subarrays by sum — especially with negative numbers, where a sliding window cannot work.

Key Idea. Range sum [L,R] = prefix[R] − prefix[L−1]. To count subarrays with sum K, store seen prefix sums in a map and look for prefix − K.
Templates

Rule #1 — Basic Prefix Sum (range queries)

Problem. Answer many range-sum queries sumRange(L, R) on a fixed array.

Input: nums = [-2,0,3,-5,2,-1], sumRange(0, 2) Output: 1 (-2 + 0 + 3)
int[] prefix = new int[n];
prefix[0] = nums[0];
for (int i = 1; i < n; i++) {
    prefix[i] = prefix[i - 1] + nums[i];
}
// sum of [L,R] = prefix[R] - (L > 0 ? prefix[L - 1] : 0)

Build O(N)  ·  Query O(1)

Rule #2 — Prefix Sum + HashMap (count sum = K)

Problem. Count the subarrays whose sum equals k (values may be negative).

Input: nums = [3,4,7,2,-3,1,4,2], k = 7 Output: 4 (has a negative — a sliding window would fail here)
Map<Integer,Integer> map = new HashMap<>();
map.put(0, 1);
int prefix = 0;
int count = 0;
for (int x : nums) {
    prefix += x;
    count += map.getOrDefault(prefix - k, 0);
    map.put(prefix, map.getOrDefault(prefix, 0) + 1);
}
return count;

Time O(N)  ·  Space O(N)

Rule #3 — Mod Prefix (sum divisible by K)

Problem. Count the subarrays whose sum is divisible by k.

Input: nums = [4,5,0,-2,-3,1], k = 5 Output: 7
Map<Integer,Integer> map = new HashMap<>();
map.put(0, 1);
int prefix = 0;
int count = 0;
for (int x : nums) {
    prefix += x;
    int rem = ((prefix % k) + k) % k;     // normalise negative remainders
    count += map.getOrDefault(rem, 0);
    map.put(rem, map.getOrDefault(rem, 0) + 1);
}
return count;

Time O(N)  ·  Space O(K)

Rule #4 — 2D Prefix Sum

Problem. Answer sum queries over any sub-rectangle of a matrix.

Input: matrix = [[3,0,1],[2,1,0],[1,2,3]], region (0,0)-(1,1) Output: 6
// pre[i][j] = sum of rectangle (0,0)..(i-1,j-1)
pre[i][j] = mat[i-1][j-1] + pre[i-1][j] + pre[i][j-1] - pre[i-1][j-1];
// sum of (r1,c1)..(r2,c2):
int s = pre[r2+1][c2+1] - pre[r1][c2+1] - pre[r2+1][c1] + pre[r1][c1];

Build O(M·N)  ·  Query O(1)

Rule #5 — Difference Array (range updates)

Problem. Apply many range-add updates, then return the final array.

Input: n = 5, updates = [[1,3,2],[2,4,3]] Output: [0,2,5,5,3]
int[] diff = new int[n + 1];
for (int[] u : updates) {            // u = [start, end, val]
    diff[u[0]] += u[2];
    diff[u[1] + 1] -= u[2];
}
int[] res = new int[n];
res[0] = diff[0];
for (int i = 1; i < n; i++) {
    res[i] = res[i - 1] + diff[i];
}

Updates O(1) each  ·  Build O(N)

Rule #6 — Prefix × Suffix Product

Problem. Return output[i] = product of all elements except nums[i] (no division).

Input: nums = [1,2,3,4] Output: [24,12,8,6]
int[] res = new int[n];
res[0] = 1;
for (int i = 1; i < n; i++) {
    res[i] = res[i - 1] * nums[i - 1];   // prefix
}
int right = 1;
for (int i = n - 1; i >= 0; i--) {
    res[i] *= right;
    right *= nums[i];
}   // suffix
return res;

Time O(N)  ·  Space O(1) extra

03—Binary Search

Repeatedly halve a sorted or monotonic search space, homing in on the answer in O(log N) comparisons instead of scanning everything.

When to use

Interview Signal

The data is sorted or monotonic, or the phrase minimize the maximum / maximize the minimum appears — a hint to binary search the answer, not an array.

Templates

Rule #1 — Classic Binary Search

Problem. Return the index of target in a sorted array, or -1.

Input: nums = [-1,0,3,5,9,12], target = 9 Output: 4
int left = 0;
int right = arr.length - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] < target) {
        left = mid + 1;
    } else {
        right = mid - 1;
    }
}
return -1;

Rule #2 — First / Last Occurrence

Problem. Find the first and last index of target in a sorted array with duplicates.

Input: nums = [5,7,7,8,8,10], target = 8 Output: [3,4]
// First: when found, record and keep going LEFT
if (arr[mid] == target) {
    result = mid;
    right = mid - 1;
}
// Last:  when found, record and keep going RIGHT
if (arr[mid] == target) {
    result = mid;
    left = mid + 1;
}

Rule #3 — Rotated Sorted Array

Problem. Search for target in a rotated sorted array.

Input: nums = [4,5,6,7,0,1,2], target = 0 Output: 4
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) {
        return mid;
    }
    if (nums[left] <= nums[mid]) {                  // left half sorted
        if (nums[left] <= target && target < nums[mid]) {
            right = mid - 1;
        } else {
            left = mid + 1;
        }
    } else {                                        // right half sorted
        if (nums[mid] < target && target <= nums[right]) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
}

Rule #4 — Find Peak Element

Problem. Return the index of any peak element (greater than its neighbours).

Input: nums = [1,2,1,3,5,6,4] Output: 5 (index of peak 6; multiple local peaks)
while (left < right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] < nums[mid + 1]) {
        left = mid + 1;   // peak on the right
    } else {
        right = mid;
    }
}
return left;

Rule #5 — Binary Search on the Answer

Problem. Minimum eating speed so Koko finishes all banana piles within h hours.

Input: piles = [3,6,7,11], h = 8 Output: 4
int minEatingSpeed(int[] piles, int h) {
    int minSpeed = 1;

    // Find max pile size
    int maxSpeed = 0;
    for (int pile : piles) {
        maxSpeed = Math.max(maxSpeed, pile);
    }

    // Binary search
    while (minSpeed < maxSpeed) {
        int mid = minSpeed + (maxSpeed - minSpeed) / 2;

        if (canEatInTime(piles, h, mid)) {
            maxSpeed = mid;
        } else {
            minSpeed = mid + 1;
        }
    }

    return minSpeed;
}

private boolean canEatInTime(int[] piles, int h, int speed) {
    int hours = 0;
    int n = piles.length;
    for (int i = 0; i < n; i++) {
        hours += (int) Math.ceil((double) piles[i] / speed);
    }

    return hours <= h;
}

Time O(N · log range)  ·  Space O(1)

Why maxSpeed = mid and not mid − 1?

This is a boundary search, not an exact-match search. A feasible mid might itself be the smallest answer, so we keep it. Only an infeasible mid is safe to discard with minSpeed = mid + 1.

Rule of thumb: drop mid only on the side where it can never be the answer.

Match the loop form to the update — mixing them infinite-loops:

  • while (lo < hi) → hi = mid  — boundary
  • while (lo ≤ hi) → hi = mid − 1  — exact match

while (left <= right)Find an exact element
while (left < right)Find a boundary / condition

Rule #6 — Search a 2D Matrix

Problem. Search a target in a fully sorted matrix (each row sorted & row[i][0] > row[i−1][last]).

Input: target = 3 matrix = ┌────┬────┬────┬────┐ │ 1 │ 3 │ 5 │ 7 │ ├────┼────┼────┼────┤ │ 10 │ 11 │ 16 │ 20 │ ├────┼────┼────┼────┤ │ 23 │ 30 │ 34 │ 60 │ └────┴────┴────┴────┘ Output: true
boolean searchMatrix(int[][] matrix, int target) {
    int m = matrix.length, cols = matrix[0].length;

    // STEP 1: find the candidate row by its [first..last] range
    int top = 0, bottom = m - 1, row = -1;
    while (top <= bottom) {
        int mid = top + (bottom - top) / 2;
        if (matrix[mid][0] <= target && target <= matrix[mid][cols - 1]) {
            row = mid; break;               // target's range is this row
        } else if (target < matrix[mid][0]) {
            bottom = mid - 1;
        } else {
            top = mid + 1;
        }
    }
    if (row == -1) return false;

    // STEP 2: binary search inside that row
    return binarySearch(matrix[row], target);
}

boolean binarySearch(int[] arr, int target) {
    int low = 0, high = arr.length - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (arr[mid] == target) return true;
        else if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return false;
}
Common mistakes

✗ Overflow — always mid = left + (right − left) / 2.

✗ Failing to spot the monotonic property for "search on answer".

✗ Mixing up minimize (right = mid) and maximize (left = mid with ceil mid).

04—Monotonic Stack

A stack kept in increasing or decreasing order, where each element is pushed and popped at most once — finding the next greater or smaller element in a single linear pass.

When to use

  • Next / previous greater or smaller element
  • Largest rectangle in a histogram
  • Sliding-window maximum (monotonic deque)
  • Building the optimal number (remove K digits)

Interview Signal

"For each element, find the next bigger/smaller one", or anything about spans, histograms, or temperatures.

Rule #1 — Next Greater Element (decreasing stack)

Problem. For each day, how many days until a warmer temperature (0 if none).

Input: temps = [73,74,75,71,69,72,76,73] Output: [1,1,4,2,1,1,0,0]
int[] result = new int[n];
Arrays.fill(result, -1);
Stack<Integer> stack = new Stack<>();        // stores indices
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && arr[stack.peek()] < arr[i]) {
        int idx = stack.pop();
        result[idx] = i - idx;    // Daily Temperatures: days until warmer
        // result[idx] = arr[i];  // Next Greater Element: the greater VALUE
    }
    stack.push(i);
}

Rule #2 — Largest Rectangle in Histogram

Problem. Largest rectangle area in a histogram of bar heights.

Input: heights = [2,1,5,6,2,3] Output: 10
Stack<Integer> stack = new Stack<>();
int maxArea = 0;
for (int i = 0; i <= n; i++) {
    int cur = (i == n) ? 0 : heights[i];
    while (!stack.isEmpty() && heights[stack.peek()] > cur) {
        int h = heights[stack.pop()];
        int w = stack.isEmpty() ? i : i - stack.peek() - 1;
        maxArea = Math.max(maxArea, h * w);
    }
    stack.push(i);
}

Rule #3 — Sliding Window Maximum (monotonic deque)

Problem. Maximum of every contiguous window of size k.

Input: nums = [1,3,-1,-3,5,3,6,7], k = 3 Output: [3,3,5,5,6,7]
Deque<Integer> dq = new ArrayDeque<>();
int[] result = new int[n - k + 1];
for (int i = 0; i < n; i++) {
    if (!dq.isEmpty() && dq.peekFirst() < i - k + 1) {
        dq.pollFirst();   // drop out-of-window
    }
    while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) {
        dq.pollLast(); // keep decreasing
    }
    dq.offerLast(i);
    if (i >= k - 1) {
        result[i - k + 1] = nums[dq.peekFirst()];
    }
}
return result;

Time O(N)  ·  Space O(K)

Rule #2 — Slow / Fast (Floyd)

Problem. Detect whether a linked list has a cycle.

Input: head = [3,2,0,-4], tail connects to index 1 Output: true
ListNode slow = head;
ListNode fast = head;
while (fast != null && fast.next != null) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow == fast) {
        return true;        // cycle found
    }
}

06—Greedy

Make the locally optimal choice at each step and never look back — valid when a greedy choice provably leads to the global optimum.

When to use

  • Interval scheduling — max non-overlapping, merge
  • Jump game · gas station · stock buy/sell
  • Partition labels · task scheduler
  • Fractional knapsack (items can be split)

Interview Signal

"Maximum / minimum / optimal" with a natural ordering, and once you commit to a choice you never need to revisit it.

Classic Problem

Non-Overlapping Intervals (Activity Selection)

Sort by end time, then greedily keep each interval that starts after the last kept end
💡
Key Idea. Sort by the right metric (often end time), then commit to every compatible choice. Finishing early leaves the most room for what follows.
Templates

Rule #1 — Non-Overlapping Intervals (sort by END)

Problem. Minimum number of intervals to remove so the rest don't overlap.

Rule #9 — Fractional Knapsack

Problem. Maximum value within a weight capacity; items may be split.

Input: capacity = 50, items(w,v) = [[10,60],[20,100],[30,120]] Output: 240.0
Arrays.sort(items, (a, b) ->
    Double.compare((double) b[1] / b[0], (double) a[1] / a[0]));   // ratio desc
double value = 0;
int rem = capacity;
for (int[] it : items) {
    if (it[0] <= rem) {
        value += it[1];
        rem -= it[0];
    } else {
        value += it[1] * ((double) rem / it[0]);
        break;
    }
}

07—Recursion & Backtracking

Solve a problem by reducing it to smaller copies of itself; backtracking adds an "undo" step so every valid configuration can be explored.

When to use

  • Generate all subsets / combinations / permutations
  • Constraint puzzles — N-Queens, word search
  • Divide and conquer — merge sort

Interview Signal

"Generate all", "find every combination", or a search through choices with dead-ends to recover from.

Classic Problem

Subsets

Subsets of [1, 2, 3] — at each element, branch exclude (left) or include (right)
💡
Key Idea. Choose → Explore → Unchoose. Define the base case first, make a move, recurse, then undo the move and try the next option.
Templates

Rule #1 — Subsets (include or exclude)

Problem. Generate all subsets of a set of distinct integers.

Input: nums = [1,2,3] Output: [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]
void subsets(int[] nums, int i, List<Integer> cur, List<List<Integer>> res) {
    if (i == nums.length) {
        res.add(new ArrayList<>(cur));
        return;
    }
    subsets(nums, i + 1, cur, res);            // exclude
    cur.add(nums[i]);
    subsets(nums, i + 1, cur, res);            // include
    cur.remove(cur.size() - 1);                // backtrack
}

Time O(2ⁿ)  ·  Space O(N)

Rule #2 — Permutations (visited array)

Problem. Generate all permutations of distinct integers.

Input: nums = [1,2,3] Output: 6 perms: [1,2,3],[1,3,2], ... ,[3,2,1]
void permute(int[] nums, List<Integer> cur, boolean[] used, List<List<Integer>> res) {
    if (cur.size() == nums.length) {
        res.add(new ArrayList<>(cur));
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) {
            continue;
        }
        used[i] = true;
        cur.add(nums[i]);
        permute(nums, cur, used, res);
        cur.remove(cur.size() - 1);
        used[i] = false;
    }
}

Time O(N! · N)  ·  Space O(N)

Rule #3 — Combinations (choose K from N)

Problem. All ways to choose k numbers from 1..n.

Input: n = 4, k = 2 Output: [[1,2],[1,3],[1,4],[2,3],[2,4],[3,4]]
void combine(int n, int k, int start, List<Integer> cur, List<List<Integer>> res) {
    if (cur.size() == k) {
        res.add(new ArrayList<>(cur));
        return;
    }
    for (int i = start; i <= n; i++) {
        cur.add(i);
        combine(n, k, i + 1, cur, res);   // i + 1: forward only
        cur.remove(cur.size() - 1);
    }
}

Rule #5 — Divide and Conquer (split → solve → merge)

Problem. Sort an array using merge sort.

Input: arr = [5,2,4,1,3] Output: [1,2,3,4,5]
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) {
        return;
    }
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

08—Top-K / Heap

A heap of size K keeps just the most important K elements live, giving O(N log K) selection without ever sorting the whole dataset.

When to use

  • Kth largest / smallest element
  • K most frequent elements
  • K closest points
  • Merge K sorted lists · streaming data

Interview Signal

You need a handful of top (or bottom) results, not a complete ordering — sorting everything would be wasteful.

Classic Problem

Top K Frequent Elements

Min-heap of size K = 3, keyed by frequency — root is the least frequent of the keepers
💡
Key Idea. For K largest use a min-heap of size K (root = smallest of the keepers, so it's first to evict). For K smallest use a max-heap.
Templates

Rule #1 — K Largest Elements (min-heap of size K)

Problem. Find the kth largest element in an array.

Input: nums = [3,2,1,5,6,4], k = 2 Output: 5
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
for (int num : nums) {
    minHeap.offer(num);
    if (minHeap.size() > k) {
        minHeap.poll();   // drop smallest
    }
}
return minHeap.peek();    // Kth largest

09—Binary Trees

Recursive structures explored depth-first (recursion / stack) or breadth-first (queue) — most tree problems reduce to picking the right traversal.

When to use

  • Traversals · level-order
  • Height · diameter · balance checks
  • Path sums · lowest common ancestor
  • BST validation & Kth smallest

Interview Signal

Anything phrased per-level (use BFS) or "process children before the parent" (postorder DFS).

Key Idea. DFS for path and structure questions; BFS with a queue (snapshot queue.size() per level) for level-based questions.

Rule #2 — Level Order (BFS)

Problem. Return node values level by level.

Input: root = [3,9,20,null,null,15,7] Output: [[3],[9,20],[15,7]]
tree for [3, 9, 20, null, null, 15, 7]

    return result;
}

q.offer(root);
while (!q.isEmpty()) {
    int size = q.size();              // nodes on this level
    List<Integer> level = new ArrayList<>();   // one list per level
    for (int i = 0; i < size; i++) {
        TreeNode node = q.poll();
        level.add(node.val);          // add the node to THIS level's list
        if (node.left != null) {
            q.offer(node.left);
        }
        if (node.right != null) {
            q.offer(node.right);
        }
    }
    result.add(level);                // level done -> push it onto the result
}
return result;

Rule #3 — Height / Diameter

Problem. Maximum depth (height) of a binary tree.

Input: root = [3,9,20,null,null,15,7] Output: 3
tree for [3, 9, 20, null, null, 15, 7] — height = 3 levels
// DFS (recursion)
int height(TreeNode root) {
    if (root == null) {
        return 0;
    }
    return 1 + Math.max(height(root.left), height(root.right));
}
// diameter: at each node, update global max with leftHeight + rightHeight
// ─────────────────────────────────────────────
// BFS (queue) — count levels
int height(TreeNode root) {
    if (root == null) {
        return 0;
    }
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    int levels = 0;
    while (!q.isEmpty()) {
        int size = q.size();              // nodes on this level
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            if (node.left != null) {
                q.offer(node.left);
            }
            if (node.right != null) {
                q.offer(node.right);
            }
        }
        levels++;                         // one full level processed
    }
    return levels;
}

Rule #4 — Same Tree / Symmetric

Problem. Is the tree a mirror image of itself?

Input: root = [1,2,2,null,3,null,3] Output: false (mirror mismatch — tests the asymmetric branch)
tree for [1, 2, 2, null, 3, null, 3] — both 3's are right children, so not a mirror
// DFS (recursion)
boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) {
        return true;
    }
    if (p == null || q == null) {
        return false;
    }
    return p.val == q.val
        && isSameTree(p.left, q.left)
        && isSameTree(p.right, q.right);
}
// symmetric: compare (left.left, right.right) and (left.right, right.left)
// ─────────────────────────────────────────────
// BFS (queue) — make a level-order list of BOTH trees (keep nulls), then compare
boolean isSameTree(TreeNode p, TreeNode q) {
    return levelOrder(p).equals(levelOrder(q));
}

List<Integer> levelOrder(TreeNode root) {
    List<Integer> out = new ArrayList<>();
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        TreeNode node = queue.poll();
        if (node == null) {
            out.add(null);                // keep nulls to preserve the SHAPE
            continue;
        }
        out.add(node.val);
        queue.offer(node.left);
        queue.offer(node.right);
    }
    return out;
}
// the nulls are essential — without them [1,2] and [1,null,2] would look equal

Rule #5 — Path Sum / Max Path Sum

Problem. Is there a root-to-leaf path summing to target?

Input: root = [5,4,8,11,null,13,4,7,2], target = 22 Output: true
tree for [5, 4, 8, 11, null, 13, 4, 7, 2] — path 5 → 4 → 11 → 2 = 22 (green)
// DFS (recursion)
boolean hasPathSum(TreeNode root, int target) {
    if (root == null) {
        return false;
    }
    if (root.left == null && root.right == null) {
        return root.val == target;
    }
    return hasPathSum(root.left, target - root.val)
        || hasPathSum(root.right, target - root.val);
}
// max path sum: gain = node.val + max(leftGain, rightGain);
// global = max(global, node.val + leftGain + rightGain)
// ─────────────────────────────────────────────
// Iterative (stack) — carry the REMAINING target with each node
boolean hasPathSum(TreeNode root, int target) {
    if (root == null) {
        return false;
    }
    Deque<TreeNode> nodes = new ArrayDeque<>();
    Deque<Integer> rem = new ArrayDeque<>();   // target left to reach at this node
    nodes.push(root);
    rem.push(target - root.val);
    while (!nodes.isEmpty()) {
        TreeNode node = nodes.pop();
        int left = rem.pop();
        if (node.left == null && node.right == null && left == 0) {
            return true;                      // leaf hit the target exactly
        }
        if (node.left != null) {
            nodes.push(node.left);
            rem.push(left - node.left.val);
        }
        if (node.right != null) {
            nodes.push(node.right);
            rem.push(left - node.right.val);
        }
    }
    return false;
}
// BFS works the same way — swap the two stacks for two queues (poll instead of pop)

Rule #6 — Lowest Common Ancestor

Problem. Lowest common ancestor of two nodes in a binary tree.

Input: root = [3,5,1,6,2,0,8], p = 5, q = 1 Output: 3
tree for [3, 5, 1, 6, 2, 0, 8] — LCA(5, 1) = 3 (green); queried nodes in amber
TreeNode lca(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) {
        return root;
    }
    TreeNode left = lca(root.left, p, q);
    TreeNode right = lca(root.right, p, q);
    if (left != null && right != null) {
        return root;   // split point
    }
    return left != null ? left : right;
}
// BST: both < root go left, both > root go right, else root is LCA

Rule #7 — BST Operations

Problem. Is the tree a valid binary search tree?

Input: root = [5,1,4,null,null,3,6] Output: false (4's left child 3 is < root 5 — only a min/max bound check catches this)
tree for [5, 1, 4, null, null, 3, 6] — 3 sits in 5's right subtree but 3 < 5 (red)
// APPROACH 1 — min/max bounds: each node must stay inside (min, max)
boolean valid(TreeNode node, long min, long max) {
    if (node == null) {
        return true;
    }
    if (node.val <= min || node.val >= max) {
        return false;
    }
    return valid(node.left, min, node.val) && valid(node.right, node.val, max);
}

// ------------------------------------------------------------------------------

// APPROACH 2 — inorder traversal: a BST's inorder is STRICTLY increasing.
// Collect inorder into a list, then verify it is sorted (no equal/decreasing pair).
List<Integer> vals = new ArrayList<>();
void inorder(TreeNode node) {
    if (node == null) return;
    inorder(node.left);
    vals.add(node.val);          // left -> node -> right
    inorder(node.right);
}
boolean isValidBST(TreeNode root) {
    inorder(root);
    for (int i = 1; i < vals.size(); i++) {
        if (vals.get(i) <= vals.get(i - 1)) return false;   // must be strictly >
    }
    return true;
}

// OPTIMIZATION — no list needed: keep only the previous value during inorder.
Integer prev = null;
boolean inorderValid(TreeNode node) {
    if (node == null) return true;
    if (!inorderValid(node.left)) return false;
    if (prev != null && node.val <= prev) return false;       // not strictly increasing
    prev = node.val;
    return inorderValid(node.right);
}

// grid DFS (islands)
void dfs(int r, int c) {
    if (r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] == '0') {
        return;
    }
    grid[r][c] = '0';                     // mark visited
    dfs(r + 1, c);
    dfs(r - 1, c);
    dfs(r, c + 1);
    dfs(r, c - 1);
}

// graph DFS (adjacency list)
void dfs(int node, boolean[] visited, List<List<Integer>> graph) {
    visited[node] = true;
    for (int nei : graph.get(node)) {
        if (!visited[nei]) {
            dfs(nei, visited, graph);
        }
    }
}

Queue<Integer> queue = new LinkedList<>();
for (int i = 0; i < n; i++) {
    if (indegree[i] == 0) {
        queue.offer(i);
    }
}
int done = 0;
while (!queue.isEmpty()) {
    int node = queue.poll();
    done++;
    for (int nei : graph.get(node)) {
        if (--indegree[nei] == 0) {
            queue.offer(nei);
        }
    }
}
return done == n;     // false if a cycle exists

Rule #4 — Union-Find (Disjoint Set Union)

Problem. Count connected components in an undirected graph.

Input: n = 5, edges = [[0,1],[1,2],[3,4]] Output: 2 (components {0,1,2} and {3,4})
int[] parent, rank;
int components;

void init(int n) {
    parent = new int[n];
    rank = new int[n];
    components = n;
    for (int i = 0; i < n; i++) parent[i] = i;   // each node is its own root
}

int find(int x) {
    while (parent[x] != x) {
        parent[x] = parent[parent[x]];   // path compression
        x = parent[x];
    }
    return x;
}

boolean union(int a, int b) {
    int ra = find(a), rb = find(b);
    if (ra == rb) return false;          // already joined -> would form a cycle
    if (rank[ra] < rank[rb]) { int t = ra; ra = rb; rb = t; }  // union by rank
    parent[rb] = ra;
    if (rank[ra] == rank[rb]) rank[ra]++;
    components--;
    return true;
}

// usage: init(n); for (int[] e : edges) union(e[0], e[1]); answer = components;
// near-O(1) per op (inverse-Ackermann) with compression + union by rank

Rule #5 — Dijkstra (weighted shortest path)

Problem. Min time for a signal from node k to reach every node (-1 if some node is unreachable).

Input: times = [[2,1,1],[2,3,1],[3,4,1]], n = 4, k = 2 Output: 2
// adj.get(u) = list of {v, weight}
int[] dist = new int[n + 1];
Arrays.fill(dist, Integer.MAX_VALUE);
dist[src] = 0;

// min-heap ordered by distance so far: {node, dist}
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[1] - b[1]);
pq.offer(new int[]{src, 0});

while (!pq.isEmpty()) {
    int[] cur = pq.poll();
    int u = cur[0], d = cur[1];
    if (d > dist[u]) continue;           // stale entry -> skip
    for (int[] edge : adj.get(u)) {
        int v = edge[0], w = edge[1];
        if (dist[u] + w < dist[v]) {      // relax the edge
            dist[v] = dist[u] + w;
            pq.offer(new int[]{v, dist[v]});
        }
    }
}
// dist[i] = shortest distance src -> i  (MAX_VALUE means unreachable)
// O(E log V). Needs NON-NEGATIVE weights (use Bellman-Ford if negative).

Rule #6 — Cycle Detection (directed + undirected DFS)

Problem. Does the graph contain a cycle?

Input: directed edges = [[0,1],[1,2],[2,0]] Output: true (0→1→2→0)
// DIRECTED: 3-color DFS  (0 = unvisited, 1 = in current path, 2 = done)
int[] state;
boolean hasCycleDirected(int u) {
    state[u] = 1;                        // entering: mark in-progress
    for (int v : graph.get(u)) {
        if (state[v] == 1) return true;  // back-edge to active node -> cycle
        if (state[v] == 0 && hasCycleDirected(v)) return true;
    }
    state[u] = 2;                        // leaving: fully explored
    return false;
}

// UNDIRECTED: DFS while remembering the node you came from
boolean hasCycleUndirected(int u, int parent) {
    visited[u] = true;
    for (int v : graph.get(u)) {
        if (!visited[v]) {
            if (hasCycleUndirected(v, u)) return true;
        } else if (v != parent) {
            return true;                 // visited & not the parent -> cycle
        }
    }
    return false;
}

Rule #1 — Linear DP (Fibonacci / House Robber)

Problem. Max money robbing houses in a row without robbing two adjacent.

Input: nums = [2,7,9,3,1] Output: 12 (2 + 9 + 1)
// dp[i] = max(dp[i-1], dp[i-2] + nums[i])
int prev2 = 0;
int prev1 = 0;
for (int num : nums) {
    int cur = Math.max(prev1, prev2 + num);
    prev2 = prev1;
    prev1 = cur;
}
return prev1;

Rule #3 — 0/1 Knapsack (take or skip — RIGHT to LEFT)

Problem. Can the array be split into two subsets of equal sum?

Input: nums = [1,5,11,5] Output: true (11 = 1+5+5)
// can reach target? (Partition Equal Subset Sum)
boolean[] dp = new boolean[target + 1];
dp[0] = true;
for (int num : nums) {
    for (int j = target; j >= num; j--) {   // RIGHT to LEFT: each item once
        dp[j] = dp[j] || dp[j - num];
    }
}

// count ways (Target Sum): sum(P) = (total + target) / 2
int[] ways = new int[subsetSum + 1];
ways[0] = 1;
for (int num : nums) {
    for (int j = subsetSum; j >= num; j--) {
        ways[j] += ways[j - num];
    }
}

Rule #4 — Unbounded Knapsack (Coin Change — LEFT to RIGHT)

Problem. Fewest coins to make up the amount (-1 if impossible).

Input: coins = [1,2,5], amount = 11 Output: 3 (5+5+1)
// min coins
int[] dp = new int[amount + 1];
Arrays.fill(dp, amount + 1);
dp[0] = 0;
for (int i = 1; i <= amount; i++) {
    for (int coin : coins) {
        if (coin <= i) {
            dp[i] = Math.min(dp[i], dp[i - coin] + 1);
        }
    }
}

// count ways (coins OUTER to avoid counting orderings twice)
dp[0] = 1;
for (int coin : coins) {
    for (int i = coin; i <= amount; i++) {
        dp[i] += dp[i - coin];
    }
}
Why coins OUTER for counting? With coins outside, each amount is built in a fixed coin order, so 2+1 is never counted separately from 1+2. Trace coins=[1,2], amount=3 → only 1+1+1 and 1+2 = 2 ways. (Min-coins doesn't care about order, so its loops can be either way.)

Rule #7 — Palindrome DP

Problem. Longest palindromic substring.

12—Intervals

Sort a list of [start, end] ranges, then sweep through them once — merging, counting, or selecting overlaps in a single pass.

When to use

  • Merge or insert overlapping ranges
  • "Can attend all meetings" / detect any overlap
  • Minimum rooms / maximum concurrent events
  • Remove the fewest intervals to eliminate overlap

Interview Signal

The input is a list of [start, end] pairs and the task mentions overlap, merge, booking, rooms, or scheduling. First instinct: sort — by start to merge/sweep, by end to greedily keep the most intervals.

Classic Problem

Merge Intervals

Sort by start, then sweep — overlapping [1,3] & [2,6] merge into [1,6]
overlapping untouched result
💡
Key Idea. Sort first. Two intervals overlap when cur.start ≤ prev.end. Sweep once — sort by start to merge or count, sort by end to greedily keep the most non-overlapping intervals.
Templates

Rule #1 — Merge Intervals (sort by start, sweep)

Problem. Merge all overlapping intervals.

Input: intervals = [[1,3],[2,6],[8,10],[15,18]] Output: [[1,6],[8,10],[15,18]]
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);     // sort by start
List<int[]> res = new ArrayList<>();
for (int[] cur : intervals) {
    int[] last = res.isEmpty() ? null : res.get(res.size() - 1);
    if (last == null || last[1] < cur[0]) {
        res.add(cur);                              // no overlap -> new interval
    } else {
        last[1] = Math.max(last[1], cur[1]);       // overlap -> extend the end
    }
}
return res.toArray(new int[0][]);

Time O(N log N)  ·  Space O(N)

Detect any overlap (e.g. "can attend all meetings?") is the same loop: sort by start, then if intervals[i][0] < intervals[i-1][1] return false — no merging needed.

Rule #2 — Insert Interval (three-phase sweep)

Problem. Insert a new interval into a sorted, non-overlapping list and merge as needed.

Input: intervals = [[1,3],[6,9]], newInterval = [2,5] Output: [[1,5],[6,9]]
List<int[]> res = new ArrayList<>();
int i = 0, n = intervals.length;
// 1. everything ending before the new interval starts
while (i < n && intervals[i][1] < newInterval[0]) res.add(intervals[i++]);
// 2. absorb every interval that overlaps the new one
while (i < n && intervals[i][0] <= newInterval[1]) {
    newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
    newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
    i++;
}
res.add(newInterval);
// 3. everything starting after the new interval ends
while (i < n) res.add(intervals[i++]);
return res.toArray(new int[0][]);

Time O(N) (already sorted)  ·  Space O(N)

Rule #3 — Min Rooms / Max Concurrent (sweep line)

Problem. Minimum number of conference rooms required.

Input: intervals = [[0,30],[5,10],[15,20]] Output: 2
int n = intervals.length;
int[] starts = new int[n], ends = new int[n];
for (int i = 0; i < n; i++) { starts[i] = intervals[i][0]; ends[i] = intervals[i][1]; }
Arrays.sort(starts);
Arrays.sort(ends);                                 // decouple starts from ends
int rooms = 0, max = 0, e = 0;
for (int s = 0; s < n; s++) {
    while (e < n && ends[e] <= starts[s]) {         // a meeting freed up first
        rooms--; e++;
    }
    rooms++;                                        // this meeting needs a room
    max = Math.max(max, rooms);
}
return max;

Time O(N log N)  ·  Space O(N)

Rule #4 — Min Removals to De-overlap (greedy by END)

Problem. Fewest intervals to remove so the rest are non-overlapping.

Input: intervals = [[1,2],[2,3],[3,4],[1,3]] Output: 1 (remove [1,3])
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);     // sort by END
int end = Integer.MIN_VALUE, removed = 0;
for (int[] iv : intervals) {
    if (iv[0] >= end) {
        end = iv[1];          // no overlap -> keep it, advance the boundary
    } else {
        removed++;            // overlaps -> drop the one that ends later
    }
}
return removed;

Time O(N log N)  ·  Space O(1)

Why sort by end? Keeping the interval that finishes earliest leaves the most room for the rest — the same exchange argument behind activity selection. Sorting by start can throw away a short interval in favour of a long greedy one.

Common mistakes

✗ Forgetting to sort first — every overlap test assumes sorted order.

✗ Sorting by start for greedy selection — the "keep most intervals" family needs sort by end.

✗ Boundary confusion: do touching intervals [1,2],[2,3] count as overlap? Pick < vs ≤ and stay consistent (in Rule #3, ends[e] ≤ starts[s] frees the room).
