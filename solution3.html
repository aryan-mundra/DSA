Q1 вҖ” Banana Transport Puzzle

LOGIC / REASONING

A camel is tasked with transporting 3,000 bananas to a destination 1,000 kilometers away.

Constraints:

  1. The camel can carry up to 1,000 bananas at a time.
  2. The camel consumes one banana per kilometer travelled.
  3. The camel can make multiple trips between any two points to transport bananas.

Question: What is the maximum number of bananas that can be transported to the destination, 1000 kilometers away?

Key Insight

Since the camel eats 1 banana per km and can only carry 1,000 bananas, if we try to go the full 1,000 km in one trip, we'd need 1,000 bananas just for travel, leaving us with 0 bananas at the destination.

The strategy: We need to create "relay stations" where we temporarily store bananas, reducing the total pile progressively as we make multiple trips.

Solution Strategy

We divide the journey into segments based on how many trips are needed:

  • Phase 1: With 3,000 bananas, we need 3 trips to move everything (3 Г— 1,000 = 3,000)
  • Phase 2: Once we're down to 2,000 bananas, we need 2 trips
  • Phase 3: Once we're down to 1,000 bananas or less, we make 1 final trip

Detailed Calculation

Phase 1: From 3,000 to 2,000 bananas (requires 5 one-way trips)

  • We need to make 3 forward trips and 2 return trips = 5 one-way trips total
  • Cost per km: 5 bananas (1 per trip)
  • We want to reduce from 3,000 to 2,000 bananas = need to spend 1,000 bananas
  • Distance: 1,000 Г· 5 = 200 km
  • After 200 km, we have: 3,000 - (5 Г— 200) = 2,000 bananas

Phase 2: From 2,000 to 1,000 bananas (requires 3 one-way trips)

  • We need to make 2 forward trips and 1 return trip = 3 one-way trips total
  • Cost per km: 3 bananas
  • We want to reduce from 2,000 to 1,000 bananas = need to spend 1,000 bananas
  • Distance: 1,000 Г· 3 = 333.33 km
  • After 333.33 km, we have: 2,000 - (3 Г— 333.33) = 1,000 bananas

Phase 3: Final journey with 1,000 bananas

  • Total distance covered so far: 200 + 333.33 = 533.33 km
  • Remaining distance: 1,000 - 533.33 = 466.67 km
  • Starting with 1,000 bananas, cost: 466.67 bananas
  • Bananas arriving at destination: 1,000 - 466.67 = 533.33 bananas

Step-by-Step Journey

PhaseDistance (km)TripsBananas UsedBananas Remaining
Start 0 вҖ” 0 3,000
Phase 1 0 вҶ’ 200 5 one-way 1,000 2,000
Phase 2 200 вҶ’ 533.33 3 one-way 1,000 1,000
Phase 3 533.33 вҶ’ 1,000 1 one-way 466.67 533.33
Answer: The maximum number of bananas that can be transported to the destination is 533 bananas (or 533.33 if we allow fractional bananas).

Q2 вҖ” Group Anagrams

JAVA

You are given an array of strings. Your task is to group the strings that are anagrams of each other into separate lists. Two strings are considered anagrams if they contain the same characters in the same frequencies, regardless of their order.

Input: An array of strings, e.g., ["eat", "tea", "tan", "ate", "nat", "bat"]
Output: A list of groups where each group contains strings that are anagrams of each other.
For example, Output: [["eat", "tea", "ate"], ["tan", "nat"], ["bat"]]

Intuition

The key idea: Anagrams have the same letters in different orders. If we sort the letters of each word, anagrams will produce the same sorted string.

For example:

  • "eat" вҶ’ sorted: "aet"
  • "tea" вҶ’ sorted: "aet"
  • "ate" вҶ’ sorted: "aet"

All three produce "aet", so they're anagrams!

Algorithm

  1. Create a HashMap where the key is the sorted version of a string, and the value is a list of all strings that match that sorted pattern
  2. For each word in the input:
    • Sort its characters to create a key
    • Add the original word to the list associated with that key
  3. Return all the lists (values) from the HashMap

Example walkthrough:

  • "eat" вҶ’ key "aet" вҶ’ map: {"aet": ["eat"]}
  • "tea" вҶ’ key "aet" вҶ’ map: {"aet": ["eat", "tea"]}
  • "tan" вҶ’ key "ant" вҶ’ map: {"aet": ["eat", "tea"], "ant": ["tan"]}
  • "ate" вҶ’ key "aet" вҶ’ map: {"aet": ["eat", "tea", "ate"], "ant": ["tan"]}
  • And so on...
import java.util.*;

public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    
    for (String str : strs) {
        // Convert string to char array, sort it, then back to string
        char[] chars = str.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars);
        
        // Add the original string to the list for this key
        if (!map.containsKey(key)) {
            map.put(key, new ArrayList<>());
        }
        map.get(key).add(str);
    }
    
    // Return all the grouped lists
    return new ArrayList<>(map.values());
}

// Alternative: Using computeIfAbsent for cleaner code
public List<List<String>> groupAnagramsV2(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    
    for (String str : strs) {
        char[] chars = str.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars);
        
        // computeIfAbsent creates the list if key doesn't exist
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(str);
    }
    
    return new ArrayList<>(map.values());
}
Time Complexity: O(n Г— k log k) вҖ” where n = number of strings, k = average length of strings (sorting each string takes k log k)
Space Complexity: O(n Г— k) вҖ” storing all strings in the HashMap

Alternative: Character Frequency Key (Optimization)

Instead of sorting, we can create a frequency-based key. This improves time complexity to O(n Г— k):

public List<List<String>> groupAnagramsOptimized(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    
    for (String str : strs) {
        // Create frequency array for 26 letters
        int[] count = new int[26];
        for (char c : str.toCharArray()) {
            count[c - 'a']++;
        }
        
        // Convert count array to string key: "1#0#0#1#1#..." 
        StringBuilder keyBuilder = new StringBuilder();
        for (int i = 0; i < 26; i++) {
            keyBuilder.append(count[i]).append('#');
        }
        String key = keyBuilder.toString();
        
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(str);
    }
    
    return new ArrayList<>(map.values());
}
Time Complexity: O(n Г— k) вҖ” where n = number of strings, k = average length of strings
Space Complexity: O(n Г— k)

Q3 вҖ” Valid Sudoku Board

JAVA

You are given a partially filled 9x9 Sudoku board. Your task is to determine if the board is valid. A Sudoku board is valid if:

  1. Each row contains the digits 1-9 without repetition.
  2. Each column contains the digits 1-9 without repetition.
  3. Each of the 9 subgrids (3x3) contains the digits 1-9 without repetition.

Only the filled cells (non-"." cells) need to be validated. Empty cells (".") can be ignored.

Input: A 9x9 grid, where each cell contains a digit from '1' to '9' or a dot (".") representing an empty cell.
Output: Return true if the board is valid according to the rules. Return false if the board violates any of the rules.

Intuition

The challenge: We need to check three things simultaneously: no duplicate in any row, any column, or any 3Г—3 sub-box.

Key insight: Use HashSets to track which numbers we've seen in each row, column, and box. If we encounter a duplicate, return false immediately.

Algorithm Strategy

  1. Create three arrays of HashSets:
    • rows[9] вҖ” one set for each row to track digits
    • cols[9] вҖ” one set for each column to track digits
    • boxes[9] вҖ” one set for each 3Г—3 box to track digits
  2. Iterate through each cell (i, j) in the board
  3. If the cell is not empty:
    • Calculate which box it belongs to: boxIndex = (i / 3) * 3 + (j / 3)
    • Check if the digit already exists in rows[i], cols[j], or boxes[boxIndex]
    • If yes, return false (duplicate found)
    • If no, add the digit to all three sets
  4. If we complete the loop without finding duplicates, return true

Understanding Box Index Calculation

The 9 boxes are numbered 0-8 in row-major order:

Box Layout:

Formula: boxIndex = (row / 3) * 3 + (col / 3)

  • Cell (0,0) вҶ’ box 0: (0/3)*3 + (0/3) = 0
  • Cell (1,4) вҶ’ box 1: (1/3)*3 + (4/3) = 0 + 1 = 1
  • Cell (5,7) вҶ’ box 5: (5/3)*3 + (7/3) = 3 + 2 = 5
  • Cell (8,8) вҶ’ box 8: (8/3)*3 + (8/3) = 6 + 2 = 8
import java.util.*;

public boolean isValidSudoku(char[][] board) {
    // Create arrays of HashSets for rows, columns, and boxes
    Set<Character>[] rows = new HashSet[9];
    Set<Character>[] cols = new HashSet[9];
    Set<Character>[] boxes = new HashSet[9];
    
    // Initialize all sets
    for (int i = 0; i < 9; i++) {
        rows[i] = new HashSet<>();
        cols[i] = new HashSet<>();
        boxes[i] = new HashSet<>();
    }
    
    // Iterate through each cell
    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < 9; j++) {
            char num = board[i][j];
            
            // Skip empty cells
            if (num == '.') {
                continue;
            }
            
            // Calculate which box this cell belongs to
            int boxIndex = (i / 3) * 3 + (j / 3);
            
            // Check if number already exists in row, column, or box
            if (rows[i].contains(num) || 
                cols[j].contains(num) || 
                boxes[boxIndex].contains(num)) {
                return false;  // Duplicate found!
            }
            
            // Add number to all three sets
            rows[i].add(num);
            cols[j].add(num);
            boxes[boxIndex].add(num);
        }
    }
    
    // No duplicates found, board is valid
    return true;
}
Time Complexity: O(1) вҖ” technically O(81) since we always check 9Г—9 = 81 cells, which is constant
Space Complexity: O(1) вҖ” we use 27 sets (9 rows + 9 cols + 9 boxes), but each can hold at most 9 elements, so it's O(27 Г— 9) = O(243) = O(1)

Alternative: Single Set with Encoded Keys

A more compact approach uses string encoding to check all three conditions with one set:

public boolean isValidSudokuCompact(char[][] board) {
    Set<String> seen = new HashSet<>();
    
    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < 9; j++) {
            char num = board[i][j];
            if (num == '.') continue;
            
            // Create unique strings for row, column, and box
            String rowKey = num + " in row " + i;
            String colKey = num + " in col " + j;
            String boxKey = num + " in box " + (i/3) + "-" + (j/3);
            
            // If any key already exists, duplicate found
            if (!seen.add(rowKey) || !seen.add(colKey) || !seen.add(boxKey)) {
                return false;
            }
        }
    }
    
    return true;
}

How the Encoding Works

For cell (4, 7) with value '5', we create three unique strings:

  • "5 in row 4" вҖ” ensures 5 appears only once in row 4
  • "5 in col 7" вҖ” ensures 5 appears only once in column 7
  • "5 in box 1-2" вҖ” ensures 5 appears only once in box (1,2)

The add() method returns false if the element already exists, allowing us to detect duplicates immediately.

Q4 вҖ” 6-Digit Palindromes

JAVA

Write a program to print all 6-digit palindromes. A palindrome is a number that reads the same forward and backward.

Examples: 100001, 101101, 123321, 999999

Intuition

Key insight: For a 6-digit palindrome, the structure is: ABCCBA

The first 3 digits determine the last 3 digits completely. So we only need to generate the first half!

  • If first half is 100, the palindrome is 100001
  • If first half is 123, the palindrome is 123321
  • If first half is 999, the palindrome is 999999

Algorithm

  1. The first 3 digits range from 100 to 999 (to ensure we get 6-digit numbers)
  2. For each number in this range:
    • Reverse it to get the second half
    • Concatenate: first half + reversed half
  3. Print the result

Total palindromes: 999 - 100 + 1 = 900 palindromes

public class SixDigitPalindromes {
    
    public static void printAllPalindromes() {
        // First half ranges from 100 to 999
        for (int i = 100; i <= 999; i++) {
            // Reverse the first half to create second half
            String firstHalf = String.valueOf(i);
            String secondHalf = new StringBuilder(firstHalf).reverse().toString();
            
            // Combine to form palindrome
            String palindrome = firstHalf + secondHalf;
            
            System.out.println(palindrome);
        }
    }
    
    public static void main(String[] args) {
        printAllPalindromes();
    }
}

// Alternative: More efficient using integer operations
public static void printAllPalindromesOptimized() {
    for (int i = 100; i <= 999; i++) {
        // Reverse the number mathematically
        int reversed = reverseNumber(i);
        
        // Combine: first half * 1000 + reversed
        int palindrome = i * 1000 + reversed;
        
        System.out.println(palindrome);
    }
}

private static int reverseNumber(int num) {
    int reversed = 0;
    while (num > 0) {
        reversed = reversed * 10 + num % 10;
        num /= 10;
    }
    return reversed;
}
Time Complexity: O(900) = O(1) вҖ” constant number of iterations
Space Complexity: O(1)

Q5 вҖ” Minimum Window Substring

JAVA

Given two strings s and t, return the minimum window in s which contains all the characters in t.

Input:
s = "ADOBECODEBANC"
t = "ABC"

Output: "BANC"

Explanation: The minimum window substring "BANC" includes 'A', 'B', and 'C' from string t.

Intuition

The problem: We need to find the smallest substring of s that contains all characters from t (with their frequencies).

Key insight: Use the sliding window technique with two pointers:

  • Right pointer: Expands the window to include more characters
  • Left pointer: Contracts the window to find the minimum size

Algorithm (Sliding Window)

  1. Setup:
    • Create a frequency map for characters in t
    • Track how many unique characters from t we've satisfied
  2. Expand (move right pointer):
    • Add characters to the window
    • Update character counts
    • When a character's count matches the required count, increment "formed" counter
  3. Contract (move left pointer):
    • Once all characters are satisfied, try to shrink the window from the left
    • Update the minimum window if current window is smaller
    • Remove characters from the left until the window becomes invalid
  4. Return the minimum window found

Visual Example

Step 1: Expand right until we have all chars

Window: "ADOBEC" вң“ (has A, B, C) вҖ” length 6

      ^     ^
    left  right

Step 3: Continue expanding right...

      ^               ^
    left            right

                    ^ ^ ^
                  left  right
Window: "BANC" вң“ (has A, B, C) вҖ” length 4 (MINIMUM!)
    
import java.util.*;

public String minWindow(String s, String t) {
    if (s.length() == 0 || t.length() == 0) {
        return "";
    }
    
    // Build frequency map for string t
    Map<Character, Integer> targetMap = new HashMap<>();
    for (char c : t.toCharArray()) {
        targetMap.put(c, targetMap.getOrDefault(c, 0) + 1);
    }
    
    int required = targetMap.size(); // Number of unique chars in t
    int formed = 0; // How many unique chars in current window match frequency
    
    // Window character counts
    Map<Character, Integer> windowMap = new HashMap<>();
    
    // Result: [window length, left, right]
    int[] result = {-1, 0, 0}; // -1 means no valid window found yet
    
    int left = 0, right = 0;
    
    while (right < s.length()) {
        // Expand window by adding character at right
        char c = s.charAt(right);
        windowMap.put(c, windowMap.getOrDefault(c, 0) + 1);
        
        // Check if frequency of current char matches target frequency
        if (targetMap.containsKey(c) && 
            windowMap.get(c).intValue() == targetMap.get(c).intValue()) {
            formed++;
        }
        
        // Contract window from left while condition is satisfied
        while (left <= right && formed == required) {
            // Update result if this window is smaller
            if (result[0] == -1 || right - left + 1 < result[0]) {
                result[0] = right - left + 1;
                result[1] = left;
                result[2] = right;
            }
            
            // Remove character at left from window
            char leftChar = s.charAt(left);
            windowMap.put(leftChar, windowMap.get(leftChar) - 1);
            
            // If this character is in t and frequency no longer matches
            if (targetMap.containsKey(leftChar) && 
                windowMap.get(leftChar) < targetMap.get(leftChar)) {
                formed--;
            }
            
            left++; // Move left pointer forward
        }
        
        right++; // Move right pointer forward
    }
    
    // Return the minimum window or empty string if not found
    return result[0] == -1 ? "" : s.substring(result[1], result[2] + 1);
}
Time Complexity: O(|s| + |t|) вҖ” each character in s is visited at most twice (once by right pointer, once by left pointer)
Space Complexity: O(|s| + |t|) вҖ” for the HashMaps

Example Walkthrough

Key Points to Remember

  • Sliding window is perfect for substring/subarray problems
  • Use two pointers: right expands, left contracts
  • Track character frequencies with HashMaps
  • The formed variable efficiently tracks when all requirements are met
  • Always update the result before breaking the valid window condition

Q6 вҖ” Student Club Management вҖ” Database Schema & SQL

SQL

You are hired to design a database for a university's student club management system. The university has multiple clubs, each with a unique club name and a president. Students can join one or more clubs, and every student has a unique student ID, name, and enrollment year. Each club organizes multiple events, with details such as the event name, event date, and club hosting the event.

Database Design Requirements

  1. Design the database schema by identifying necessary tables, their attributes, and relationships (primary key and foreign keys)
  2. Make sure that the schema is in the 3NF.
  3. Answer the following SQL queries once the schema is created.

Understanding the Problem

What we're building: A database to manage student clubs at a university. We need to track:

  • Which students belong to which clubs (many-to-many: one student can join multiple clubs, one club has multiple students)
  • Who is the president of each club
  • Which events each club organizes
  • Which students attended which events (many-to-many: one student can attend multiple events, one event has multiple attendees)

Why 5 Tables? (Breaking Down the Design)

  1. students вҖ” Core table storing basic student info (ID, name, enrollment year)
  2. clubs вҖ” Core table storing club info (name, who's president)
    • President is a foreign key to students table (president must be a valid student)
  3. student_clubs вҖ” Junction table connecting students вҶ” clubs
    • Why needed? A student can join many clubs, and a club has many members
    • Each row represents one membership: (student_id, club_id)
  4. events вҖ” Stores each event organized by a club
    • Each event belongs to exactly one club (foreign key to clubs)
  5. event_participants вҖ” Junction table connecting students вҶ” events
    • Why needed? A student can attend many events, and an event can have many attendees
    • Each row represents one attendance: (event_id, student_id)

What is 3NF? (Simple Explanation)

Third Normal Form (3NF) means our database has no redundancy and data is organized properly:

  • Rule 1: Every column depends on the primary key (no unrelated data in a table)
  • Rule 2: No transitive dependencies (A вҶ’ B вҶ’ C, where B is not a key)
  • Example of violation: If we stored "club_name" in the events table, we'd repeat the same club name for every event. Instead, we store only club_id and look up the name when needed.

Our design is in 3NF because each table stores only directly related information, and we use foreign keys to connect related data without duplication.

CREATE TABLE students (
    student_id      INT          PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,
    enrollment_year INT          NOT NULL
);

CREATE TABLE clubs (
    club_id      INT          PRIMARY KEY AUTO_INCREMENT,
    club_name    VARCHAR(100) NOT NULL UNIQUE,
    president_id INT          NOT NULL,
    FOREIGN KEY (president_id) REFERENCES students(student_id)
);

CREATE TABLE student_clubs (
    student_id INT NOT NULL,
    club_id    INT NOT NULL,
    PRIMARY KEY (student_id, club_id),
    FOREIGN KEY (student_id) REFERENCES students(student_id),
    FOREIGN KEY (club_id)    REFERENCES clubs(club_id)
);

CREATE TABLE events (
    event_id   INT          PRIMARY KEY AUTO_INCREMENT,
    event_name VARCHAR(200) NOT NULL,
    event_date DATE         NOT NULL,
    club_id    INT          NOT NULL,
    FOREIGN KEY (club_id) REFERENCES clubs(club_id)
);

CREATE TABLE event_participants (
    event_id   INT NOT NULL,
    student_id INT NOT NULL,
    PRIMARY KEY (event_id, student_id),
    FOREIGN KEY (event_id)   REFERENCES events(event_id),
    FOREIGN KEY (student_id) REFERENCES students(student_id)
);

SQL Query 1 вҖ” List all students along with the clubs they are a part of

Requirement: If a student is not part of any club, still include their name in the output.

SELECT s.student_id,
       s.name        AS student_name,
       c.club_name
FROM   students s
LEFT JOIN student_clubs sc ON s.student_id = sc.student_id
LEFT JOIN clubs c          ON sc.club_id = c.club_id
ORDER BY s.name, c.club_name;

Explanation

  • LEFT JOIN ensures all students appear, even if they're not in any club
  • Students with no clubs will have NULL for club_name
  • Students in multiple clubs will appear multiple times (one row per club)

SQL Query 2 вҖ” Find the name of the club president and the number of events organized by their club in the year 2024

SELECT s.name          AS president_name,
       c.club_name,
       COUNT(e.event_id) AS events_in_2024
FROM   clubs c
JOIN   students s ON c.president_id = s.student_id
LEFT JOIN events e ON c.club_id = e.club_id
                   AND YEAR(e.event_date) = 2024
GROUP BY c.club_id, c.club_name, s.name
ORDER BY events_in_2024 DESC;

Explanation

  • JOIN students to get president name from president_id
  • LEFT JOIN events to include clubs with zero events in 2024
  • Filter year 2024 in the JOIN condition (not WHERE) to preserve clubs with 0 events
  • COUNT(e.event_id) counts events; returns 0 for clubs with no matching events
  • GROUP BY groups by club (and president) to aggregate event counts

SQL Query 3 вҖ” Retrieve the details of all events (event name, date, and club name) along with the names of students who participated in them

Requirement: Include events that no student attended.

SELECT e.event_name,
       e.event_date,
       c.club_name,
       s.name AS participant_name
FROM   events e
JOIN   clubs c              ON e.club_id = c.club_id
LEFT JOIN event_participants ep ON e.event_id = ep.event_id
LEFT JOIN students s           ON ep.student_id = s.student_id
ORDER BY e.event_date, e.event_name, s.name;

Explanation

  • JOIN clubs to get the club name for each event
  • LEFT JOIN event_participants ensures events with no participants are included
  • LEFT JOIN students to get participant names
  • Events with no participants will have NULL for participant_name
  • Events with multiple participants will appear multiple times (one row per participant)
